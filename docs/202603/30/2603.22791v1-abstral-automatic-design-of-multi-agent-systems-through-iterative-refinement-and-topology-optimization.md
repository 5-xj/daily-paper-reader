## 摘要
多智能体系统（MAS）应如何设计？这些设计知识能否以一种可检查、可修订且可迁移的形式被捕获？我们提出了 ABSTRAL，这是一个将 MAS 架构视为不断演进的自然语言文档的框架，该文档是通过对比轨迹分析（contrastive trace analysis）进行细化的产物。研究得出了三项发现：首先，我们对多智能体协调成本（coordination tax）进行了精确测量：在固定轮次预算下，集成系统仅达到 26% 的轮次效率，66% 的任务耗尽了限制，但通过发现可并行的任务分解，其表现仍优于单智能体基准。其次，编码在文档中的设计知识具有可迁移性：在一个领域学习到的拓扑推理和角色模板为新领域提供了先发优势，迁移后的种子在单次迭代中即可达到冷启动第 3 次迭代的性能。第三，对比轨迹分析能够发现初始设计中不存在的专家角色，这是以往系统所不具备的能力。在 SOPBench（包含 134 个银行任务和确定性预言机）上，以 GPT-4o 为骨干网络的 ABSTRAL 达到了 70% 的验证集通过率和 65.96% 的测试集通过率。我们发布了收敛后的文档，作为可检查的设计原理。

## Abstract
How should multi-agent systems be designed, and can that design knowledge be captured in a form that is inspectable, revisable, and transferable? We introduce ABSTRAL, a framework that treats MAS architecture as an evolving natural-language document, an artifact refined through contrastive trace analysis. Three findings emerge. First, we provide a precise measurement of the multi-agent coordination tax: under fixed turn budgets, ensembles achieve only 26% turn efficiency, with 66% of tasks exhausting the limit, yet still improve over single-agent baselines by discovering parallelizable task decompositions. Second, design knowledge encoded in documents transfers: topology reasoning and role templates learned on one domain provide a head start on new domains, with transferred seeds matching coldstart iteration 3 performance in a single iteration. Third, contrastive trace analysis discovers specialist roles absent from any initial design, a capability no prior system demonstrates. On SOPBench (134 bank tasks, deterministic oracle), ABSTRAL reaches 70% validation / 65.96% test pass rate with a GPT-4o backbone. We release the converged documents as inspectable design rationale.

#### 2. 方法论：ABSTRAL 框架
ABSTRAL 采用三层架构循环优化 MAS 设计：

*   **核心工件 (SKILL.md)**：一个结构化的 Markdown 文档，包含四个部分：
    *   **K (领域知识)**：任务相关的特定事实。
    *   **R (拓扑推理)**：条件规则（如“若任务具有属性 P，则倾向于拓扑 T”）。
    *   **T (角色模板)**：自动发现的专家角色及其系统提示词、工具权限。
    *   **P (构建协议)**：构建可运行智能体图的步骤。
*   **内层循环（轨迹驱动细化）**：
    1.  **Build**：元智能体根据 SKILL.md 生成智能体规范并实例化系统。
    2.  **Run**：在沙盒环境中执行任务，捕获全量执行轨迹。
    3.  **Analyze**：将失败与成功轨迹配对，归类为 5 种证据类别（EC1-EC5，涵盖知识错误、路由瓶颈、角色缺失、接口不匹配等）。
    4.  **Update**：根据证据对 SKILL.md 的特定章节进行针对性编辑。
*   **整合层（防止语义漂移）**：每 5 次迭代进行规则合并与精简，去除冗余或低频规则。
*   **外层循环（拓扑多样性）**：利用**图编辑距离（GED）**和语义余弦距离作为排斥约束，强制系统探索完全不同的拓扑家族（如从层级制转向集成制），防止陷入局部最优。

#### 6. 主要结论与发现
*   **协调税的量化**：多智能体系统在固定轮次预算下，轮次效率仅为 26%，66% 的任务会触及轮次上限。MAS 的优势不在于步数节省，而在于发现**可并行的任务分解**。

*   **性能表现**：在 SOPBench 银行任务上，ABSTRAL 达到了 70% 的验证通过率和 65.96% 的测试通过率，优于 GPT-4o 单智能体基准。

#### 7. 优点
*   **高可解释性**：优化目标是自然语言文档，设计决策（为什么选这个拓扑、为什么要这个角色）对人类透明且可审计。
*   **拓扑感知优化**：不同于仅优化 Prompt 的方法，ABSTRAL 能在宏观架构层面进行搜索。
*   **证据驱动**：通过 5 种明确的证据类别将失败原因与文档章节挂钩，使优化过程更具因果性而非随机搜索。

#### 8. 不足与局限
*   **轮次预算敏感**：由于多智能体路由消耗了大量轮次，在极短轮次限制的任务中，MAS 表现可能不如单智能体。

*   **搜索方差**：由于验证集采样较小，不同运行批次之间存在约 5% 的性能波动。
*   **分类体系限制**：目前的拓扑分类（7 类）和证据分类（5 类）可能无法涵盖所有复杂的失败模式。

## 摘要
autoresearch 仓库使 LLM 智能体能够通过直接编辑训练代码，在无约束的搜索空间中搜索最优超参数配置。在给定的固定计算预算和约束条件下，我们使用 autoresearch 作为测试平台，在微调小型语言模型的超参数任务上，对比了经典超参数优化（HPO）算法与基于 LLM 的方法。在固定的超参数搜索空间内，CMA-ES 和 TPE 等经典 HPO 方法的表现始终优于基于 LLM 的智能体。然而，尽管仅使用了自托管的开源 27B 模型，但在无约束搜索空间中直接编辑训练源码的 LLM 智能体大幅缩小了与经典方法之间的差距。避免显存溢出（OOM）失败的方法优于搜索多样性更高的方法，这表明可靠性比探索广度更重要。虽然中小型 LLM 难以在多次试验中追踪优化状态，但经典方法缺乏领域知识。为了弥补这一差距，我们引入了 Centaur，这是一种将 CMA-ES 的内部状态（包括均值向量、步长和协方差矩阵）与 LLM 共享的混合方法。Centaur 在我们的实验中取得了最佳结果，其 0.8B 变体的表现优于 27B 变体，这表明当与强大的经典优化器配对时，廉价的 LLM 就足够了。0.8B 模型不足以进行无约束的代码编辑，但足以进行混合优化；而在所测试的开源模型中，将模型规模扩大到 27B 对于固定搜索空间方法并没有带来优势。代码可在 https://github.com/ferreirafabio/autoresearch-automl 获取。

## Abstract
The autoresearch repository enables an LLM agent to search for optimal hyperparameter configurations on an unconstrained search space by editing the training code directly. Given a fixed compute budget and constraints, we use \emph{autoresearch} as a testbed to compare classical hyperparameter optimization (HPO) algorithms against LLM-based methods on tuning the hyperparameters of a small language model. Within a fixed hyperparameter search space, classical HPO methods such as CMA-ES and TPE consistently outperform LLM-based agents. However, an LLM agent that directly edits training source code in an unconstrained search space narrows the gap to classical methods substantially despite using only a self-hosted open-weight 27B model. Methods that avoid out-of-memory failures outperform those with higher search diversity, suggesting that reliability matters more than exploration breadth. While small and mid-sized LLMs struggle to track optimization state across trials, classical methods lack domain knowledge. To bridge this gap, we introduce Centaur, a hybrid that shares CMA-ES's internal state, including mean vector, step-size, and covariance matrix, with an LLM. Centaur achieves the best result in our experiments, with its 0.8B variant outperforming the 27B variant, suggesting that a cheap LLM suffices when paired with a strong classical optimizer. The 0.8B model is insufficient for unconstrained code editing but sufficient for hybrid optimization, while scaling to 27B provides no advantage for fixed search space methods with the open-weight models tested. Code is available at https://github.com/ferreirafabio/autoresearch-automl.

### 1. 核心问题与研究动机
*   **研究背景**：Andrej Karpathy 的 `autoresearch` 项目展示了 LLM 智能体可以通过直接编辑代码来优化模型。随后有研究指出，传统的 TPE 算法在相同预算下可能优于 LLM 智能体。
*   **核心问题**：在严格受限的计算预算和相同的实验条件下，LLM 驱动的优化方法是否真的能击败经典的 HPO 算法（如 CMA-ES、TPE）？
*   **动机**：LLM 拥有领域知识（如对 Transformer 结构的直觉），但缺乏对优化状态的精确追踪能力；经典算法擅长数值优化，但对任务背景一无所知。研究旨在探索两者的优劣并尝试结合。

### 2. 方法论
*   **自动搜索空间构建**：利用抽象语法树（AST）解析技术，从训练脚本中自动提取 14 个超参数（如学习率、Batch Size、层数等），减少人为干预。
*   **对比方法分类**：
    *   **经典 HPO**：TPE、CMA-ES、SMAC、随机搜索。
    *   **纯 LLM 方法**：Karpathy Agent（固定空间 vs. 直接编辑代码）、LLAMBO（将 LLM 作为贝叶斯优化的代理模型）。
    *   **Centaur（半人马座）混合方法**：这是本文的核心创新。它将 CMA-ES 的内部状态（均值向量 $\mu$、步长 $\sigma$、协方差矩阵 $C$）以自然语言形式提供给 LLM。在 30% 的试验中，LLM 会参考这些状态、历史最优配置和最近试验记录，对 CMA-ES 提出的建议进行“覆盖”或微调。
*   **核心思想**：让经典优化器负责全局搜索轨迹的维护，让 LLM 利用领域知识（如“Batch Size 应该是 2 的幂”或“特定深度下的注意力窗口选择”）对候选方案进行启发式修正。

### 3. 实验设计
*   **实验场景**：优化一个约 50M 参数的小型 Transformer 模型（nanochat），在 FineWeb 数据集上训练，目标是最小化验证集的 bits-per-byte (val_bpb)。
*   **Benchmark**：以 Karpathy 的 `autoresearch` 任务为基准。
*   **对比维度**：在 24 小时固定 GPU 预算下，对比 9 种方法的收敛速度、最终性能、显存溢出（OOM）率和搜索多样性。

### 4. 资源与算力
*   **硬件**：使用单张 **NVIDIA H200 GPU** (141 GB VRAM)。
*   **算力分配**：为确保公平，将 80GB 显存分配给模型训练，剩余显存用于自托管 LLM（使用 vLLM 部署 Qwen3.5-27B 或 0.8B）。
*   **时长**：每种方法运行 3 个随机种子，每个种子分配 **24 小时** 连续训练时间。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 9 种算法，每种算法 3 个种子，总计进行了数百次模型训练试验。
*   **消融实验**：
    *   **模型规模**：对比了 0.8B 和 27B 模型作为优化器的差异。
    *   **混合比例**：测试了 Centaur 中 LLM 参与比例（$r$）对结果的影响。
    *   **模型能力**：初步对比了开源模型 Qwen3.5 与闭源模型 Gemini 2.5 Flash。
*   **评价**：实验设计非常严谨，控制了墙钟时间（Wall-clock time）而非仅仅是试验次数，考虑了 LLM 推理的开销，具有很高的客观性和公平性。

### 6. 主要结论与发现
*   **经典算法依然强劲**：在固定的超参数空间内，TPE 和 CMA-ES 的表现始终优于纯 LLM 方法。
*   **LLM 的优势在于“无约束”**：当 LLM 被允许直接编辑代码（而非受限于固定参数范围）时，其表现大幅提升，接近经典算法。
*   **可靠性胜过多样性**：LLM 经常提出导致 OOM 的配置（失败率高达 40%-60%），而经典算法能更快学会避开非法区域。
*   **Centaur 混合法最优**：Centaur 结合了 CMA-ES 的稳定性和 LLM 的启发式知识，取得了全场最佳结果。
*   **模型规模的悖论**：对于代码编辑，27B 模型远强于 0.8B；但在 Centaur 混合模式下，**0.8B 模型的表现甚至优于 27B**，说明在有强大优化器辅助时，极小的模型也足以提供有效的领域启发。

### 7. 优点
*   **实战导向**：直接在真实的训练代码库上进行测试，而非简单的数学基准函数。
*   **混合范式**：Centaur 提供了一种优雅的“白盒”协作方式，将优化器的数学状态显式传给 LLM，而非仅仅传历史分数。
*   **工程严谨**：自动提取超参数和严格的显存/时间控制，消除了很多 AutoML 论文中常见的“人工调优优化器”的偏差。

### 8. 不足与局限
*   **任务单一**：仅在 `nanochat` 一个任务上进行了验证，尚未证明在更大规模模型或不同领域（如 CV）的泛化性。
*   **搜索空间限制**：虽然自动提取了参数，但参数的取值范围（Range）仍需人工指定，这依然引入了人类先验。
*   **模型局限**：主要测试了开源中小型模型，最顶尖的闭源模型（如 GPT-4o, Claude 3.5）在代码编辑模式下是否能彻底击败经典算法仍待观察。

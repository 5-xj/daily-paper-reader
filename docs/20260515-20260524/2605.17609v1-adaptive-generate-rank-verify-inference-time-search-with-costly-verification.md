## 摘要
许多推理时语言模型流水线将廉价的奖励信号与昂贵的验证器结合使用，例如数学推理中的精确答案检查或代码生成中的隐藏测试执行。我们通过学习理论的视角将此设置形式化为生成式主动搜索：一个成本敏感的首个正样本搜索问题，其中策略自适应地从未知分布中采样候选，观察廉价分数，并支付验证器标签直到找到正样本。对于固定提示，生成器和奖励模型产生两个未知对象：奖励分数分布和基于分数的成功函数。当这些量已知时，我们使用动态规划方法刻画分布感知的最优策略。在分数分布和成功函数均未知的现实实用场景中，我们提出ADAP，一种逐层自适应生成-排序-验证算法，逐步增加采样响应和排名靠前的验证数量。在单调性假设下（即更高的奖励分数同样可能通过验证），我们证明ADAP的期望成本在常数因子内接近分布感知最优。我们辅以基于中心星数的学习理论下界，表明分数-标签关系的结构假设是必要的。在数学推理和竞技编程上的实验验证了相对于固定非自适应策略和难度自适应基线的预期优势。

## Abstract
Many inference-time language-model pipelines combine a cheap reward signal with an expensive verifier, such as exact answer checking in mathematical reasoning or hidden-test execution in code generation.   We formalize this setting using a learning-theoretic lens as generative active search: a cost-sensitive first-positive search problem in which a policy adaptively samples candidates from an unknown distribution, observes cheap scores, and pays for verifier labels until it finds a positive example. For a fixed prompt, the generator and reward model induce two unknown objects: a distribution over reward scores and a score-conditioned success function. When these quantities are known, we characterize the distribution-aware optimal policy using a dynamic programming approach. In the realistic and practical setting where both the score distribution and success function are unknown, we propose ADAP, a shellwise adaptive generate-rank-verify algorithm that progressively increases the number of sampled responses and top-ranked verifications. Under the monotonicity assumption that higher reward scores are no less likely to pass verification, we show that ADAP achieves expected cost within a constant factor of the distribution-aware optimum. We complement this result with learning-theoretic lower bounds, based on a centered star number, showing that structural assumptions on the score--label relationship are necessary. Experiments on mathematical reasoning and competitive programming validate the predicted advantage over both fixed non-adaptive policies and difficulty-adaptive baselines.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题背景**：现代大语言模型（LLM）推理时，常采用“生成-排序-验证”流水线：首先廉价生成多个候选回答，用奖励模型（reward model）快速打分排序，最后仅对少数高分候选进行昂贵的验证（例如，数学推理中的精确答案匹配，代码生成中的隐藏测试执行）。这种模式中，生成、评分和验证都会消耗计算资源（时间、GPU、API费用）。
- **核心矛盾**：每个提示的难度差异极大，简单提示只需少量生成和一次验证，困难提示则需要大量生成和多次验证。固定预算策略（固定生成数N_rew和验证数N_ver）要么在简单提示上浪费成本，要么在困难提示上搜索不足，无法适应提示难度的变化。
- **研究目标**：设计一种**在线、自适应、提示级**的策略，仅利用当前提示的奖励分数和验证结果，在未知生成分布和未知验证成功率函数的情况下，实现接近“已知最优分布策略”的成本效率。
- **理论意义**：揭示了该问题中必须对奖励分数与验证成功之间的关系施加结构假设（如单调性），否则任何策略都无法达到常数倍最优。

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

#### 2.1 问题形式化：主动搜索（Active Search）
- 定义**主动搜索实例**：对于固定提示 \(x\)，生成器诱导分数分布 \(D_x\)，条件成功函数 \(h^*_x(r)=\mathbb{P}(V=1|R=r)\)（奖励分数 \(r\) 时通过验证的概率）。二者均未知。
- 策略成本：\(J = c_{\text{rew}} N_{\text{rew}} + c_{\text{ver}} N_{\text{ver}}\)，其中 \(c_{\text{rew}}\) 为生成和评分的单次成本，\(c_{\text{ver}}\) 为验证单次成本。策略必须找到第一个通过验证的样本，否则成本无限大。

#### 2.2 分布感知最优基准（Theoretical Benchmark）
- **定理3.2**：若已知 \(D_x\) 和 \(h^*_x\)，最优策略为**阈值策略**：验证仅当 \(h^*(r) > \tau^*\)，其中 \(\tau^*\) 由方程 \(c_{\text{ver}} \mathbb{E}[\max\{h^*(R)-\tau^*,0\}] = \tau^* c_{\text{rew}}\) 唯一确定。最优期望成本 \(J^* = c_{\text{ver}}/\tau^*\)。

#### 2.3 ADAP算法：自适应策略
- **核心假设（单调性）**：\(h^*\) 是单调非减函数，即更高奖励分数至少具有相同或更高的通过概率。这保证了奖励模型排序是有信息量的。
- **算法流程（Shellwise Active Search）**：
  1. **分层搜索**：定义多个“壳”（shell），每个壳 \(s\) 包含一对整数 \((a,b)\) 满足成本 \(B_{a,b}=c_{\text{rew}}2^b + c_{\text{ver}}2^{b-a}\) 在 \([2^s c_{\min}, 2^{s+1} c_{\min})\) 区间内，\(c_{\min}=\min\{c_{\text{rew}}, c_{\text{ver}}\}\)。
  2. 在每个壳 \(s\)，选择最大的样本尺度 \(b_s^*\) 和最大的验证尺度 \(j_s^*\)，生成 \(m_s = \lceil 2^{b_s^*+1}\rceil\) 个新候选，并允许验证 \(k_s = \lceil 6\cdot 2^{j_s^*}\rceil\) 个。
  3. 维护一个未验证候选池，每次生成后重新按奖励分数降序排列，验证池中排名前 \(k_s\) 且未验证的候选。一旦某次验证成功，立即输出并停止。
  4. 壳的规模逐级递增，从而总成本被控制在常数因子内。

- **理论保证（定理4.2）**：在单调性假设下，ADAP的期望成本不超过分布感知最优成本的400倍（常数因子）。常数来自壳间几何级数代价的加和。

#### 2.4 学习理论基础
- **中心星数（Centered Star Number）** \(s_0(\mathcal{H})\)：用于刻画无结构假设时主动搜索的难度。定义为存在最多多少个点 \(z_1,...,z_m\) 和对应概念 \(h_1,...,h_m\) 使得 \(h_i(z_j)=1\) 当且仅当 \(i=j\)，且所有概念中包括全零函数。
- **定理6.2（下界）**：若 \(s_0(\mathcal{H})\) 很大（如无限），则任何策略在最坏情况下的期望成本至少是分布感知最优成本的 \(\frac14\min\{s_0, c_{\text{ver}}/c_{\text{rew}}\}\) 倍，说明无结构假设时不可能有统一常数界。
- **定理6.3（匹配上界）**：给出算法ACS，在二进制概念类下达到 \(O(\min\{s_0, c_{\text{ver}}/c_{\text{rew}}\})\) 的竞争比，与下界匹配。

### 8. 不足与局限

- **单调性假设是否始终成立？** 虽然在实验中验证了平均情况下排序正确率较高，但部分问题奖励模型可能不可靠（如数学AUC中位数0.577，73%高于随机，仍有约27%提示排序不优于随机）。这些情况下ADAP可能退化为随机验证。
- **仅处理可行提示**：论文假设每个提示至少存在一个正确候选，未讨论完全失败的情况（如所有样本皆错），此时ADAP会无限成本。实际中可能需要提前停止策略。
- **成本模型简化**：将生成和验证成本视为固定标量，未考虑响应长度、并行生成带来的开销差异。
- **仅使用奖励分数作为唯一信号**：忽略了对未验证响应的内容分析（如语法错误等）。未来可整合更丰富的特征。
- **实验规模有限**：仅测试了小模型（3B、7B），在更大模型（如70B）或更复杂任务上的泛化性未知。数据集规模也有限（数学22题、编程83题）。
- **常数因子较大（400×）** 理论上可能保守，实际观察到的成本远低于理论界，但理论保证较弱。未来或可优化常数。
- **未与其他自适应验证方法（如Qu2026的可行性门）直接比较**，仅对比了固定策略和困难类别策略。
- **代码和奖励模型的选取局限**：仅使用了Qwen系列和CodeScaler，未测试其他主流模型（如GPT系列、DeepSeek系列等）。附录中虽有部分替代生成器，但奖励模型固定，奖励模型的泛化性未充分探讨。

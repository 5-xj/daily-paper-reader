## 摘要
基于遗传编程的符号回归（GPSR）可能会出现过拟合和结构膨胀问题，尤其是在存在噪声的情况下。本文评估了描述长度（DL）和分数贝叶斯因子（FBF）准则，作为启发式方法的具有原理性且数据高效的选择替代方案，用于选择泛化能力好的紧凑表达式。我们使用基于Fisher信息的参数编码实现DL，并在多个数据集上将其与AIC和BIC进行比较，包括带噪声的合成基准测试和真实世界的回归问题。我们研究了三种搜索/选择策略：（i）多目标搜索准确性和程序长度，随后进行DL/FBF选择；（ii）直接将DL作为目标的多目标搜索；（iii）以DL/FBF作为适应度的单目标优化。在多个数据集上，我们发现与AIC/BIC基线相比，DL/FBF后选择提高了测试性能，并且BIC结合与DL/FBF相同的函数复杂度惩罚产生了类似的结果。相比之下，在单目标GPSR中直接使用DL/FBF作为适应度函数经常导致过早收敛到过于简单的模型。最后，我们提供了关于在遗传编程工作流中使用DL/FBF作为稳健模型选择工具的实用指导。

## Abstract
Symbolic regression with genetic programming (GPSR) may suffer from overfitting and structural bloat, especially when noise is present. In this paper we evaluate description length (DL) and fractional Bayes factor (FBF) criteria as principled, data-efficient alternatives to heuristics for selecting compact expressions that generalise well. We implement DL using a Fisher-information-based parameter encoding and compare it to AIC and BIC across multiple datasets, including noisy synthetic benchmarks and real-world regression problems. We study three search/selection strategies: (i) multi-objective search for accuracy and program length followed by DL/FBF selection; (ii) multi-objective search using DL directly as an objective; and (iii) single-objective optimisation with DL/FBF as the fitness. Across datasets we find that DL/FBF post-selection improves test performance compared to AIC/BIC baseline and that BIC in combination with the same function complexity penalty from DL/FBF produces similar results. In contrast, using DL/FBF directly as a fitness function in single-objective GPSR frequently induces premature convergence to overly simple models. We conclude with practical guidance for using DL/FBF as robust model-selection tools in genetic programming workflows.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 基于MDL原则，将模型描述长度分解为数据编码长度（负对数似然）加上模型编码长度（结构复杂度 + 参数复杂度）。
- 结构复杂度：用表达式长度 * 符号集大小对数加上常数编码长度表示。
- 参数复杂度：通过Fisher信息矩阵的奇异值分解（SVD）计算参数所需精度对应的编码长度，避免传统AIC/BIC只依赖参数个数的缺点。
- FBF为DL的贝叶斯变体，避免了参数整数离散化引入的不恰当先验，使用数据的一小部分作为校准样本。

### 关键技术细节
- **DL公式**（含参数复杂度旋转后版本）：
  \( DL = -\log L + k\log n + \sum \log c_i + \frac12\sum \max(0,\log S_{ii} - \log 3 + \log|\hat{\theta}_i^{\mathrm{rot}}|) \)
  其中 \(k\) 为树节点数，\(n\) 为不同符号数，\(c_i\) 为常量，\(S_{ii}\) 为Fisher信息矩阵的奇异值。
- **FBF公式**（不含参数复杂度项，使用分数先验）：
  \( FBF = -(1-b)\log L + \text{函数复杂度} + \frac{p}{2}(\log(2\pi e^{1-\log 3}) - \log b) \)
  其中 \(b=m^{-1/2}\)，\(m\) 为样本数。
- **BIC_SR**：将标准BIC与函数复杂度项结合： \( BIC_{SR} = -2\log L + 2\ \text{函数复杂度} + p\log m \)。
- 所有准则均基于高斯似然，参数通过Levenberg-Marquardt优化，噪声方差 \(\sigma_{\text{err}}^2\) 由残差MSE估计。

### 算法流程
论文测试三种搜索/选择策略：
1. **MO-Length**：多目标优化（NSGA-II），目标为NLL和程序长度；在每一代帕累托前沿中，使用不同准则（AIC/BIC/BIC_SR/DL/FBF）选择一个模型作为该代最佳。
2. **MO-DL**：多目标优化，目标为NLL和DL的惩罚项（函数复杂度+参数复杂度），最终用DL选择。
3. **SO**：单目标GPSR（类似Koza风格），直接以AIC/BIC/BIC_SR/DL/FBF作为适应度函数最小化。

所有变体均对每个候选表达式进行参数优化（memetic优化），并使用相同的GP算子（子树交叉、突变）。

## 6. 论文的主要结论与发现

1. **多目标搜索 + DL/FBF/BIC_SR后选择显著优于AIC和BIC**。在测试MSE和泛化能力上，DL、FBF、BIC_SR效果接近，远好于AIC和BIC，后者容易过拟合。

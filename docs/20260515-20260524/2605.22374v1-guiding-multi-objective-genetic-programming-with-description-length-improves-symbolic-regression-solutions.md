## 摘要
基于遗传规划的符号回归可能出现过拟合和结构膨胀，尤其在存在噪声时。本文评估了描述长度（DL）和分数贝叶斯因子（FBF）准则作为原则性、数据高效的替代启发式方法，用于选择泛化能力好的紧凑表达式。我们使用基于Fisher信息的参数编码实现DL，并在多个数据集（包括含噪声的合成基准和真实回归问题）上将其与AIC和BIC进行比较。我们研究了三种搜索/选择策略：（i）多目标搜索（精度和程序长度）后跟DL/FBF选择；（ii）直接使用DL作为目标的多目标搜索；以及（iii）以DL/FBF为适应度的单目标优化。在多个数据集上，我们发现与AIC/BIC基线相比，DL/FBF后选择提升了测试性能，且BIC结合来自DL/FBF的相同函数复杂度惩罚产生了类似结果。相比之下，在单目标GPSR中直接使用DL/FBF作为适应度函数经常导致过早收敛到过于简单的模型。最后，我们给出了在遗传规划工作流中使用DL/FBF作为鲁棒模型选择工具的实用指南。

## Abstract
Symbolic regression with genetic programming (GPSR) may suffer from overfitting and structural bloat, especially when noise is present. In this paper we evaluate description length (DL) and fractional Bayes factor (FBF) criteria as principled, data-efficient alternatives to heuristics for selecting compact expressions that generalise well. We implement DL using a Fisher-information-based parameter encoding and compare it to AIC and BIC across multiple datasets, including noisy synthetic benchmarks and real-world regression problems. We study three search/selection strategies: (i) multi-objective search for accuracy and program length followed by DL/FBF selection; (ii) multi-objective search using DL directly as an objective; and (iii) single-objective optimisation with DL/FBF as the fitness. Across datasets we find that DL/FBF post-selection improves test performance compared to AIC/BIC baseline and that BIC in combination with the same function complexity penalty from DL/FBF produces similar results. In contrast, using DL/FBF directly as a fitness function in single-objective GPSR frequently induces premature convergence to overly simple models. We conclude with practical guidance for using DL/FBF as robust model-selection tools in genetic programming workflows.

### 核心思想
- 在遗传规划符号回归中，将模型选择准则（DL、FBF）与多目标或单目标优化结合，引导搜索更紧凑、泛化更好的表达式。
- DL基于Fisher信息矩阵和奇异值分解计算参数精度编码，FEF使用分数贝叶斯因子避免不当先验。

### 关键技术细节
- **模型选择准则**：
  - **负对数似然**：假设i.i.d.高斯噪声，NLL公式为 \(-\log L = \frac{m}{2}\log(2\pi\sigma^2) + \frac{1}{2\sigma^2}\sum (y_i - f(x_i,\theta))^2\)，参数通过Levenberg-Marquardt算法优化。
  - **AIC**：\(AIC = -2\log L + 2p\)（p为参数个数）。
  - **BIC**：\(BIC = -2\log L + p\log m\)。
  - **BIC SR**（扩展BIC加入函数复杂度惩罚）：\(BIC_{SR} = -2\log L + 2\cdot\text{func\_comp} + p\log n\)。
  - **DL**：\(DL = -\log L + \text{func\_comp} + \text{param\_comp}\)，其中函数复杂度\(\text{func\_comp} = k\log n + \sum \log c_i\)，参数复杂度\(\text{param\_comp} = \frac{1}{2}\sum \max(0,\log S_{ii} - \log 3 + \log|\hat{\theta}^{\text{rot}}_i|)\)，\(S_{ii}\)为Fisher信息矩阵奇异值。
  - **FBF**：\(FBF = -(1-b)\log L + \text{func\_comp} + \frac{p}{2}(\log(2\pi e^{1-\log 3}) - \log b)\)，\(b = m^{-1/2}\)。
- **算法变体**：
  - **MO-Length**：多目标优化（NLL和程序长度），使用NSGA-II，再在帕累托前沿中用各准则选最优模型。
  - **MO-DL**：多目标优化（NLL和DL的惩罚项，即函数复杂度+参数复杂度）作为第二目标。
  - **SO**：单目标优化，直接使用AIC、BIC、BIC SR、DL或FBF作为适应度。
- **参数优化**：每个候选表达式通过memetic优化（Levenberg-Marquardt）拟合参数，更新表达式。
- **DL计算改进**：使用完整Fisher信息矩阵（FIM）的SVD分解，处理强相关参数，O(p³)复杂度。

## 6. 论文的主要结论与发现
- **多目标后选择策略最优**：在MO-Length后使用DL、FBF或BIC SR选择模型，所得表达式紧凑、测试误差低，与AIC/BIC相比显著改善（DL vs AIC胜率88.8%，DL vs BIC胜率73.0%）。

- **MDL自动适应噪声和样本量**：低噪声时选更长表达式，高噪声时选更短表达式；小样本时偏好短表达式（有时导致欠拟合）。
- **推荐**：对于GPSR，建议使用DL、FBF或BIC SR作为后选择工具搭配多目标搜索（NLL+长度），避免直接单目标优化；BIC SR和FBF计算成本更低，效果类似。

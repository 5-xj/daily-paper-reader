## 摘要
列表式重排序是基于视觉的检索和多模态检索增强生成（M-RAG）处理长文档时的关键但计算昂贵的组件。虽然近期基于VLM的重排序器取得了很高的准确率，但其实用性常受到长视觉令牌序列和多步自回归解码的限制。我们提出ZipRerank，一种高效处理两个瓶颈的列表式多模态重排序器。它通过轻量级的查询-图像早期交互机制减少输入长度，并通过单次前向传播对所有候选进行评分，从而消除自回归解码。为了有效学习，ZipRerank采用两阶段训练策略：(i) 在大规模渲染为图像的文本数据上进行列表式预训练，以及(ii) 使用VLM教师蒸馏的软排序监督进行多模态微调。在MMDocIR基准上的大量实验表明，ZipRerank在匹配或超越最先进的多模态重排序器的同时，将LLM推理延迟降低了一个数量级，使其非常适合对延迟敏感的实时系统。代码可在 https://github.com/dukesun99/ZipRerank 获取。

## Abstract
Listwise reranking is a key yet computationally expensive component in vision-centric retrieval and multimodal retrieval-augmented generation (M-RAG) over long documents. While recent VLM-based rerankers achieve strong accuracy, their practicality is often limited by long visual-token sequences and multi-step autoregressive decoding. We propose ZipRerank, a highly efficient listwise multimodal reranker that directly addresses both bottlenecks. It reduces input length via a lightweight query-image early interaction mechanism and eliminates autoregressive decoding by scoring all candidates in a single forward pass. To enable effective learning, ZipRerank adopts a two-stage training strategy: (i) listwise pretraining on large-scale text data rendered as images, and (ii) multimodal finetuning with VLM-teacher-distilled soft-ranking supervision. Extensive experiments on the MMDocIR benchmark show that ZipRerank matches or surpasses state-of-the-art multimodal rerankers while reducing LLM inference latency by up to an order of magnitude, making it well-suited for latency-sensitive real-world systems. The code is available at https://github.com/dukesun99/ZipRerank.

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- **ZipRerank**：一个高度高效的列表式多模态重排序框架，直接针对两个延迟瓶颈。
- **训练**：两阶段策略——第一阶段在大规模文本数据（渲染为图像）上预训练通用重排序能力；第二阶段利用更强的 VLM 教师（GPT-5）生成软排序标签，进行多模态微调。
- **推理**：① 通过轻量级**查询-图像早期交互**机制进行视觉 token 剪枝，保留最相关的 token；② 采用**单 token 解码**（single-logit scoring），在一次前向传播中为所有候选打分，消除自回归解码。

### 2.2 关键技术细节

#### 训练阶段
- **第一阶段（通用重排序预训练）**：
  - 使用 RankZephyr 文本数据集，将文本段落渲染为 280×280 图像。
  - 优化目标：`L_stage1 = LLM + λ1 * L_ranknet`。
  - `L_ranknet`：加权 RankNet 损失，基于标识符 logits 的成对排序对数损失，权重与排名成反比。
  - `LLM`：标准语言建模交叉熵损失，预测标识符序列。
- **第二阶段（视觉重排序微调）**：
  - 使用 MMDocIR 训练集（VQA 风格数据），仅提供单条正例，由 GPT-5 教师生成完整候选排序作为软标签。
  - 优化目标：`L_stage2 = LLM + λ2 * L_softrank`。
  - `L_softrank`：软排序损失，基于几何衰减的列表级交叉熵。教师排序位置 k 的目标概率 `q_π(k) = γ^k / Σ`，γ∈(0,1)，灵感来自 Rank-Biased Precision。

#### 推理阶段
- **查询-图像早期交互**：
  - 将查询 token 化，仅运行 LLM 的前缀部分，提取查询 token 的隐藏状态 `H_q`。
  - 对每页图像 i 的视觉 token 嵌入 `v_i,j`，计算与所有查询 token 的余弦相似度，取最大值作为重要性分数 `a_i,j`。
  - 对每页保留 top-ρ 比例的视觉 token（ρ∈(0,1]），保留原始 RoPE 位置编码。
  - 复用前缀 KV cache，剪枝开销极小。
- **单 token 解码**：
  - 每个候选页分配唯一单 token 标识符（如 A、B、...）。
  - 在格式化输入上运行一次 LLM，取下一 token 预测中标识符的 logits `z_ti`，按 logits 降序排列得到最终排序。
  - 完全避免自回归生成多步解码。

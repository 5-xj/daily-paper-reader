## 摘要
缺失模态会导致多模态推荐系统的严重故障。在冷启动场景中，用户历史、物品文本和视觉证据经常缺失，而这正是推荐质量最为关键的时刻。现有方法通过插补、特征传播或生成式重建来恢复缺失信号，但当存留信号较弱时，这些策略可能会注入无依据的证据。我们提出了元模态代理（MMA），一种基于大语言模型的候选池重排序器，将缺失性视为一个顺序证据路由问题。MMA通过平衡缺失性任务强化学习在掩码模态片段上进行训练，并在两个变体中进行评估：MMA-Auto仅使用自动化文本、图像和图形工具；MMA-Interactive额外允许基于存留模态的澄清问题，作为上界诊断。MMA在第一阶段检索器产生候选池后运行；它对候选进行评分，而非从完整目录中检索物品。最终的重排序将MMA分数与基于验证数据选择的第一阶段检索分数融合。我们的评估围绕四个证据检查组织，这些检查对于鲁棒的缺失模态声明是必要的：无预言机单观测模态可用性（OOMA）鲁棒性、每个模态的OOMA分解、固定池全目录重排序以及确定性路由器机制控制。MMA-Auto在目标正例OOMA NDCG@10上比最强非交互基线提升了4.0%，在固定池全目录重排序NDCG@10上提升了12.7%。RuleRouter-Fuse使用相同的工具和融合规则但无学习策略更新，其性能不及MMA-Auto，支持了超越确定性工具融合的学习路由。当提供澄清时，MMA-Interactive带来了4.1%的上界增益。

## Abstract
Missing modalities cause severe failures in multimodal recommender systems. User histories, item text, and visual evidence are frequently absent during cold-start scenarios, exactly when recommendation quality matters most. Existing approaches recover absent signals through imputation, feature propagation, or generative reconstruction, but these strategies can inject unsupported evidence when the surviving signals are weak. We introduce the Meta-Modal Agent (MMA), a large language model based candidate-pool reranker that treats missingness as a sequential evidence-routing problem. MMA is trained with balanced missingness-task reinforcement learning over masked-modality episodes and is evaluated in two variants: MMA-Auto, which uses only automated text, image, and graph tools, and MMA-Interactive, which additionally permits clarification questions grounded in surviving modalities as an upper-bound diagnostic. MMA operates after a first-stage retriever has produced a candidate pool; it scores those candidates rather than retrieving items from the full catalog. Final reranking fuses MMA scores with first-stage retrieval scores selected on validation data. Our evaluation is organized around four evidence checks required for a robust missing-modality claim: oracle-free one-observed-modality availability (OOMA) robustness, per-modality OOMA breakdowns, fixed-pool full-catalog reranking, and a deterministic-router mechanism control. MMA-Auto improves target-positive OOMA NDCG@10 by 4.0% and fixed-pool full-catalog reranking NDCG@10 by 12.7% over the strongest non-interactive baseline. RuleRouter-Fuse, which uses the same tools and fusion rule without learned policy updates, underperforms MMA-Auto, supporting learned routing beyond deterministic tool fusion. MMA-Interactive adds a 4.1% upper-bound gain when clarification is available.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：多模态推荐系统中，用户历史、物品文本和视觉证据在冷启动场景下经常缺失，导致推荐质量严重下降。
- **现有方法局限**：传统方法通过插补、特征传播或生成式重建来恢复缺失信号，但当存留信号质量弱时，这些重建可能注入不可靠的证据，反而损害推荐效果。
- **核心创新视角**：将模态缺失从静态表示学习问题重新定义为**顺序证据路由问题**——即如何动态决定调用哪个证据工具、如何处理工具返回的“空（Null）”结果，并基于存留证据对候选池进行重排序。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 提出 **Meta-Modal Agent (MMA)**，一个基于大语言模型（LLM）的候选池重排序器，不进行全目录检索，只对第一阶段检索器产生的候选项评分和重排序。
- 将缺失性建模为部分可观测马尔可夫决策过程（POMDP），代理通过调用预定义的证据工具（文本分析、图像分析、图谱检索）获取观测，并将“Null”返回视为第一类证据，驱动后续路由决策。

### 关键技术细节
1. **模块化架构**（四个模块）：
   - 模块1（Episode Interface）：固定候选池，包含目标项和硬负例，代理只能对该池评分。
   - 模块2（Evidence-Routing Agent）：LLM策略依次输出推理轨迹和JSON动作，调用证据工具。工具包括：
     - `Analyze_Text(item_id)`：返回文本字段，若文本模态缺失则返回Null。
     - `Analyze_Image(item_id)`：返回CLIP top-5语义标签，图像缺失则Null。
     - `Retrieve_Graph(user_id)`：返回协同邻居标题，行为缺失则Null。
     - `Ask_User(modality_type, query)`：仅用于交互式变体MMA-Interactive，作为上界诊断。
   - 模块3（Candidate Scoring and Fusion）：终端动作`Score_Candidates`输出每个候选的相关性分数，与第一阶段检索分数按验证集选定的权重α融合：ŝ_i = α s^(0)_i + (1-α) s^MMA_i。
   - 模块4（Balanced Missingness-Task Training）：定义7个任务族（所有模态可用、三种LOMO、三种OOMA），均匀采样训练，避免易样本主导。

2. **训练方法**：
   - 使用近端策略优化（PPO）优化期望奖励，奖励包括NDCG@10（终端）和步骤成本（工具调用−0.02，用户询问−0.10，无效动作−0.20）。
   - 基础LLM为Llama-3-8B-Instruct（8-bit量化），LoRA微调（r=16, α=32），冻结基座权重。
   - 训练时通过掩码模态制造缺失片段，使代理反复经历工具失效并学习恢复。

3. **变体**：
   - **MMA-Auto**：禁用Ask_User，仅用自动化工具（核心部署版本）。
   - **MMA-Interactive**：启用Ask_User，作为交互式上界诊断。

## 7. 优点

- **问题重定义新颖**：将缺失模态从“恢复问题”转换为“路由问题”，使“Null”返回成为有价值的观测，而非失败。
- **部署友好的架构**：MMA仅对候选池评分，不参与全目录检索，降低搜索空间，可无缝挂接任意第一阶段检索器。
- **严格的实验分离**：明确区分自动化变体（MMA-Auto）与交互式上界（MMA-Interactive），避免夸大部署承诺。
- **充分的控制实验**：通过RuleRouter-Fuse严格控制工具访问和融合规则，量化学习路由的增益。
- **统计严谨**：报告配对显著性检验和效应量，避免小样本波动下的过度解释。
- **跨数据集一致性**：在三种不同证据配置（Amazon结构化文本+商品图、Yelp评论+场地照片）下获得一致改进，验证泛化性。

## 8. 不足与局限

- **计算成本高**：代理推理比静态基线慢约100倍（1.250s vs 0.012s），在实时推荐场景中可能不可行。论文承认存在准确率-延迟权衡。
- **实验场景有限**：仅评估三个公开数据集，未覆盖视频、音频等模态，也未在真实A/B测试中验证。
- **交互式上界理想化**：Ask_User工具回应由模型从存留证据合成，非真实用户反馈，低估了实际交互噪声和成本。
- **冷启动假设**：预设候选池已由第一阶段检索器提供，未研究第一阶段本身失效的情况（如检索未召回目标）。
- **方法复杂性**：需要LLM部署、PPO训练、LoRA微调、工具调用环境，工程成本高于传统嵌入方法。
- **泛化担忧**：仅在单一基础LLM（Llama-3-8B）上测试，未知对其他LLM（如GPT-4、LLaMA-70B）的可迁移性。
- **未讨论偏见与公平性**：代理基于工具返回的证据做出路由决策，可能加剧已有模态偏差；未分析对弱势用户（如无行为历史用户）的公平性。

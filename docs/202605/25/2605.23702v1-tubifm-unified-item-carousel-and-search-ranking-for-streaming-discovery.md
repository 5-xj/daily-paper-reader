## 摘要
个性化发现系统通常为条目排序、轮播排序和搜索训练独立的模型，尽管这些任务从相同的观众旅程中暴露互补信号：观看行为塑造轮播和条目排序，搜索查询即使未导致目录匹配也揭示意图，而观看历史有助于将搜索解释为重新观看、续看或新发现。我们引入了用户故事（user story），这是一种序列化表示，将用户的跨表面历史——属性、会话、带有表面和轮播上下文的观看事件以及搜索事件——转换为单一令牌序列。通过将预训练语言令牌与领域特定事件令牌交错，用户故事使异构的推荐和搜索任务能够被表达为基于共享语法的提示式下一个令牌预测。TubiFM 是这种方法的一个实例：一个基于 Llama 3.2 1B 的模型，在用户故事上训练，并被提示对条目、轮播或搜索结果进行排序，无需任务特定架构。在离线评估中，这个单一模型在条目、轮播和搜索排序上均优于专业基线。在在线A/B测试中，TubiFM 显著提升了搜索总观看时间（TVT）+3.9%，轮播 TVT +0.30%。条目排序在 TVT 上统计中性（+0.14%），但匹配了成熟的生产栈；在所有三个任务中，TubiFM 在 L40S GPU 上运行，并将 p99 排序延迟从 500ms 降低到 200ms。这些结果表明，共享的用户故事可以改进发现，同时简化排序系统。

## Abstract
Personalized discovery systems often train separate models for item ranking, carousel ranking, and search, even though these tasks expose complementary signals from the same viewer journey: watches shape carousel and item ranking, search queries reveal intent even when they do not lead to a catalog match, and watch history helps interpret search as rewatching, continuation, or new discovery. We introduce the user story, a serialized representation that turns a user's cross-surface history - attributes, sessions, watch events with surface and carousel context, and search events - into a single token sequence. By interleaving pretrained language tokens with domain-specific event tokens, user stories let heterogeneous recommendation and search tasks be expressed as prompted next-token prediction over a shared grammar. TubiFM is one instantiation of this approach: a Llama 3.2 1B-based model trained on user stories and prompted to rank items, carousels, or search results without task-specific architectures. In offline evaluation, this single model outperforms specialist baselines across item, carousel, and search ranking. In online A/B tests, TubiFM significantly improves search total viewing time (TVT) by $+3.9\%$ and carousel TVT by $+0.30\%$. Item ranking is statistically neutral on TVT ($+0.14\%$), but matches a mature production stack; across all three tasks, TubiFM serves on L40S GPUs and reduces p99 ranking latency from 500ms to 200ms. These results show that shared user stories can improve discovery while simplifying ranking systems.

## 1. 论文的核心问题与整体含义（研究动机和背景）

流媒体平台的个性化发现系统通常**为条目排序（item ranking）、轮播排序（carousel ranking）和搜索排序（search ranking）训练独立的模型**。这种做法忽视了这些任务天然存在的**互补信号**：用户的观看行为可以改善轮播和条目排序；搜索查询即使未命中目录也能揭示意图；观看历史有助于理解搜索行为（是重新观看、续看还是新发现）。现有系统不仅增加了维护成本，也无法充分利用跨任务信息。论文的核心动机是**提出一种统一的表示和建模方法，让同一模型同时处理三种排序任务，从而简化系统并提升效果。**

## 2. 论文提出的方法论

### 核心思想：用户故事（User Story）
- 将用户的全部跨表面历史（属性、会话、带表面和轮播上下文的事件、搜索事件）序列化为**单一令牌序列**。
- 通过交错**预训练语言令牌**与**领域特定事件令牌**，使异构的推荐和搜索任务能够表达为**基于共享语法的提示式（prompted）下一个令牌预测**。

### 关键技术细节
1. **序列构造**：
   - **用户属性**：粗粒度的国家、设备等文本序列化。
   - **会话结构**：超过1小时不活跃则新会话开始，每个会话最多12小时，用 `<|session|>` 标记边界并编码时间差。
   - **事件数据**：观看事件（物品ID、观看时长、表面、轮播）、搜索事件（查询文本、搜索即输入时记录中间查询）。

2. **模型架构**：
   - 基于 **Llama 3.2 1B** 预训练检查点，扩展词表（加入领域令牌），通过自回归下一个令牌预测微调。
   - 训练语料包含用户故事（约2000万用户，平均长度560令牌）和辅助目录语料（将ID映射回文本，如物品标题、轮播名称），混合比例20:1。
   - 训练时使用**随机掩码**：10%概率将表面/轮播对替换为掩码，0.1%概率替换物品ID为未知令牌，以支持容器无关排序和目录更新。

3. **推理与任务表达**：
   - 不同任务通过附加不同的**提示头**（prompt head）实现：条目排序追加带掩码轮播的观看事件头；轮播排序追加表面上下文预测轮播令牌；搜索排序追加查询令牌和搜索表面观看头。
   - 一次前向传播打分全量目录（原子物品ID令牌，无需解码），延迟低。

4. **物品ID表示**：
   - 使用**单原子令牌**而非语义ID，因为目录规模约10万，原子ID推理成本更低，前期实验表明语义ID无额外收益。

## 3. 实验设计

### 数据集/场景
- 内部流媒体样本，约**2000万用户**，10万唯一标题，1000唯一轮播，8亿观看事件，6600万搜索事件。
- 四类表面：首页、搜索、浏览、自动播放。用户级切分：99%训练，1%离线评估。

### 对比方法（离线基准）
- **条目/轮播排序**：SASRec（自注意力序列推荐）、HSTU（时序Transformer）以及**任务特定的TubiFM变体**（相同架构但仅使用任务特定视图）。
- **搜索排序**：BM25（稀疏词汇检索）、Qwen3-4B Embeddings（零样本密集检索）、微调Sentence-Transformer（基于all-MiniLM-L6-v2，使用查询-标题正对）。

### 评估指标
- Hit Rate (HR@K) 和 Normalized Discounted Cumulative Gain (NDCG@K)，K∈{8,50,100}。

### 在线A/B测试
- 在真实生产系统上进行，每个表面运行一周，使用**观众级随机化**，主要指标为**总观看时间（TVT）**，显著性水平p<0.05（使用CURE方差缩减）。
- 控制组为当时活跃的生产栈：条目排序为多召回+DCN排序器；搜索为DCN两阶段系统；轮播为DCN模型。

## 4. 资源与算力

- **训练**：8× **H100 GPU**，序列长度1024，每设备批量4，梯度累积4，训练约**22小时**（120k步）。
- **在线推理**：使用 **L40S GPU** 和动态批处理，p99延迟从生产系统的500ms降至约**200ms**。
- 模型每日增量训练，目录令牌约每月刷新一次。

## 6. 论文的主要结论与发现

1. **单一统一模型可超越所有专用系统**：TubiFM在离线评估中大幅优于SASRec、HSTU、BM25等强基线，且统一训练比任务特定训练效果更好（如条目HR@8提升16.9%，搜索HR@8提升12.1%）。
2. **在线效果显著**：搜索TVT +3.9%（尤其长尾查询+20%），轮播TVT +0.30%（顶部位置提升明显），条目排序TVT中性（+0.14%不显著），但延迟大幅降低。
3. **延迟优势**：p99从500ms降至200ms，架构简化（无需多级召回和特征工程）。
4. **用户故事的关键组件**：会话结构、预训练初始化对性能贡献最大，用户属性影响微弱。

## 7. 优点（方法与实验设计的亮点）

- **统一表示的简洁性**：用户故事将异构行为统一为令牌序列，新增信号只需添加字段，无需重新设计架构。
- **提示式框架的灵活性**：改变提示头即可切换任务，共享训练和推理代码。
- **原子ID令牌的设计**：避免了解码开销，一次前向覆盖全目录，适合生产低延迟要求。
- **掩码训练的巧妙性**：训练时随机掩码轮播和物品ID，支持容器无关排序和目录动态更新，增强了模型鲁棒性。
- **消融实验系统全面**：覆盖初始化、属性、会话结构三个关键维度，验证了核心设计选择。
- **在线验证的严谨性**：使用观众级随机化、CURE方差缩减、一周运行周期，显著性水平严格。

## 8. 不足与局限

- **标签偏差**：所有离线标签来自隐式反馈（观看行为），非人工相关性判断，存在曝光偏差和正样本仅反馈问题。
- **在线控制组的专有性**：生产系统为内部多组件管道，无法公开复现，仅报告相对提升。
- **领域泛化有限**：实验仅在流媒体视频场景验证，用户故事框架的通用性在其他领域（如电商、音乐）未测试。
- **用户属性影响小**：模型中属性贡献微弱，可能因粗粒度或行为序列主导，但未深入分析是否可优化。
- **搜索评估不完善**：仅使用单个观看后物品作为正例，忽略多个相关结果，可能低估检索方法。
- **训练成本**：虽22小时可接受，但每日重新训练需要持续计算资源，小规模团队可能受限。

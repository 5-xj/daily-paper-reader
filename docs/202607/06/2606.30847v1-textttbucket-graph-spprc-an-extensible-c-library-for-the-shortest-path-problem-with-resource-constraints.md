## 摘要
我们提出 \texttt{bucket-graph-spprc}（简称 \texttt{bgspprc}），这是一个开源的、仅头文件的 C++23 库，用于求解带资源约束的最短路径问题（SPPRC），这是分支切割定价法在车辆路径规划及相关问题中的核心定价子问题。该库实现了 Sadykov、Uchoa 和 Pessoa（2021）的桶图标记算法，包括双向标记、跨弧拼接、桶固定和弧消除，以及采用 SIMD 加速支配检查的结构化数组（structure-of-arrays）标记存储。其核心设计特点是编译时资源概念：通过实现固定的七函数接口即可添加新的 SPPRC 变体，资源组合成标记状态时无需运行时调度，状态布局在编译时确定。内置五种资源：时间/容量、ng路径元素性松弛、秩1割、累积成本以及取送货。在一个可复现的、基于相同边界的共享公共实例上的头对头比较中，\texttt{bgspprc} 在移位几何均值上比主要开源对比库 PathWyse（Salani, Basso 和 Giuffrida, 2024）快 1.3 倍至 2.35 倍（即使自身单线程运行时也快 1.3 倍至 2.3 倍），并且运行速度约为并行拉取标记（Petersen 和 Spoorendonk, 2025）（一种针对同一问题的不同标记技术）的 1.9 倍至 2.4 倍。该库、基准测试脚本以及固定实例均公开可用。

## Abstract
We present $\texttt{bucket-graph-spprc}$ ($\texttt{bgspprc}$ for short), an open-source, header-only C++23 library for the shortest path problem with resource constraints (SPPRC), the pricing subproblem at the heart of branch-cut-and-price for vehicle routing and related problems. The library implements the bucket-graph labelling algorithm of Sadykov, Uchoa and Pessoa (2021), with bidirectional labelling, across-arc concatenation, bucket fixing and arc elimination, and a structure-of-arrays label store with SIMD-accelerated dominance. Its central design feature is a compile-time resource concept: a new SPPRC variant is added by implementing a fixed seven-function interface, and resources compose into a label state with no runtime dispatch, the state layout fixed at compile time. Five resources ship built in: time/capacity, ng-path elementarity relaxation, rank-1 cuts, cumulative cost, and pickup-and-delivery. In a reproducible, head-to-head comparison on shared public instances at an identical bound, $\texttt{bgspprc}$ outperforms PathWyse (Salani, Basso and Giuffrida, 2024), the main open-source comparator, by $1.3\times$--$2.35\times$ in shifted geometric mean (and by $1.3\times$--$2.3\times$ even when itself run single-threaded), and runs within $1.9\times$--$2.4\times$ of parallel pull labelling (Petersen and Spoorendonk, 2025), a different labelling technique for the same problem. The library, benchmark scripts, and pinned instances are publicly available.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：带资源约束的最短路径问题（SPPRC）是分支切割定价法（branch-cut-and-price）在车辆路径规划及相关问题中的核心定价子问题。SPPRC 要求在给定有向图上找到一条从源点到汇点的最小成本路径，同时满足多种资源约束（如时间窗、容量等）。求解 SPPRC 是整个列生成过程的计算瓶颈。
- **背景**：现有解法包括基于标签的动态规划算法（如双向搜索、ng-path松弛、桶图标记等），但开源实现匮乏：VRPSolver 为闭源学术软件；PathWyse 实现了标准标记但未采用先进的桶图算法；cspy 注重灵活性而非性能；BALDES 将标记嵌入完整求解器，不易提取复用。因此，社区缺少一个高性能、可扩展且开源独立的桶图 SPPRC 库。
- **研究动机**：填补这一空白，提供一个开放、可复现、高效且易扩展的 C++ 库，让研究者能够直接应用桶图算法并轻松添加新资源变体。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：基于 Sadykov, Uchoa 和 Pessoa（2021）的桶图标记算法，并采用编译时资源概念（Resource Concept），将资源的依赖操作抽象为七函数接口，通过模板元编程组合资源状态，避免运行时派发。
- **关键技术细节**：
  - **桶图算法**：将标签按照主资源值划分到桶中，按拓扑顺序处理桶；利用完成下界（completion bound）进行桶固定和弧消除，大幅剪枝搜索空间。
  - **双向标记**：同时从源点和汇点扩展前向和后向标签，在中间点（基于第一主资源的中间值 μ）通过跨弧拼接恢复完整路径。
  - **跨弧拼接**：前向和后向标签通过一条弧连接时，计算拼接成本 γ，利用资源相关函数检查兼容性。
  - **支配规则**：采用带惩罚的支配（dominance），包含主资源比较和资源依赖的惩罚项 δ，并通过 min_domination_cost 常数进行快速预检查。
  - **SIMD 加速支配**：标签采用结构体数组（SoA）存储，SIMD 向量化进行成本与主资源预过滤，以及 ng 子集测试。
  - **多阶段求解**：自动从启发式（仅保留每桶最便宜标签、忽略 ng 和秩1状态）升级到精确阶段，保留最优性保证。
- **资源概念**：任何资源类型需实现 `Resource` 概念规定的七个函数：`symmetric()`, `init_state()`, `extend_along_arc()`, `extend_to_vertex()`, `domination_cost()`, `concatenation_cost()`, `min_domination_cost()`。用户只需编写实现这些函数的类即可添加新 SPPRC 变体。
- **内置资源**：时间/容量、ng-path元素性松弛、秩1割（支持最多64个有限记忆秩1割，具有有限正向惩罚 δ）、累积成本、取送货。

## 6. 主要结论与发现
- **性能优势**：在相等 bound 下，bgspprc 比 PathWyse 快 1.3×–2.35×（SGM 比），即使自身单线程运行时也快 1.3×–2.3×。在 188 个共同求解的实例对中，bgspprc 在 176 个上更快。
- **与并行拉取标记对比**：bgspprc 运行时间为并行拉取标记的 1.9×–2.4×（SGM 比），差距随 ng 增大而缩小（从 2.4× 降至 1.9×），表明固定开销而非随搜索规模扩展。
- **消融发现**：
  - 方向（单方向 vs 双向）是主导因素：浅搜索（ng=8 的 rcspp）单方向更快；深度搜索时双向更快（up to 3.1×）。
  - 线程并行提供 1.02–1.27× 的额外加速（除个别标量 rcspp 8 悬崖外）。
  - SIMD 在纯 ng 实例上无益（甚至更慢 0.74–0.99×），预计在活跃秩1割的场景下可能有效。
- **默认模式鲁棒性**：默认 `para_bidir_vec` 虽非每条件最快，但避免了任何灾难性回归（如标量并行双向在 rcspp ng=8 超时 19/56 个实例）。

## 7. 优点

## 摘要
自进化框架通常优化任务解决方案，同时将周围的支架视为固定的。我们提出了支架感知自进化（HASE），一个智能体强化学习框架，其中单个模型可以在多轮动作空间中生成任务解决方案或编辑选定的支架组件。HASE使得单个Qwen3-8B模型能够匹配使用Claude Code作为支架提议器的GPT-OSS-120B模型的文本分类性能。在阿尔法因子挖掘方面，HASE优于报道的GPT-OSS-120B基线。HASE还能修复不完善的评估组件，并在圆填充算法发现中收敛到最先进的性能。这些结果表明，HASE通过一个统一的智能体过程改进了支架和解决方案。

## Abstract
Self-evolving frameworks usually optimize task solutions while treating the surrounding harness as fixed. We introduce Harness-Aware Self-Evolving (HASE), an agentic reinforcement-learning framework in which a single model can generate task solutions or edit selected harness components in a multi-turn action space. HASE enables a single Qwen3-8B model to match the text-classification performance of a GPT-OSS-120B model that uses Claude Code as the harness proposer. In alpha factor mining, HASE outperforms the reported GPT-OSS-120B baseline. HASE also repairs imperfect evaluation components and converges to state-of-the-art performance in circle-packing algorithm discovery. These results show that HASE improves the harness and the solution through one unified agentic process.

# Harness-Aware Self-Evolving (HASE) 论文详细总结

## 1. 核心问题与整体含义（研究动机和背景）

- **现有缺陷**：当前自进化系统（如 FunSearch、AlphaEvolve、Meta-Harness 等）通常将任务周围的“支架”（harness）视为固定不变。Harness 包括提示词、记忆、检索策略、工具接口、评估器等组件，它们决定了模型观测什么、能做什么、如何被评分。固定 harness 的假设会导致两个问题：① 人工编写的评估器可能不完整或被错误对齐，模型在分数驱动优化下会利用这些漏洞（reward hacking）；② 即使评估器正确，引导 harness 也可能提供较差的搜索先验（如冗余记忆、缺失有用结构等）。
- **研究动机**：让同一个模型能够同时改进任务解决方案和自身所使用的 harness 组件，从而突破固定 harness 的限制，实现模型、任务方案和 harness 的协同进化。
- **核心贡献**：提出 Harness-Aware Self-Evolving（HASE），一种智能体强化学习框架，允许单一模型在多轮行动空间中既生成解决方案又编辑选定的 harness 组件。无需依赖外部强大代码提议器（如 Claude Code），即可在多个任务上达到或超越大模型系统性能。

## 2. 方法论：核心思想、关键技术细节、算法流程

- **核心思想**：将 harness 作为可进化的部分，与模型权重和任务方案一起优化。HASE 区分两种 harness：
  - **引导组件**（Guidance Components）：包括提示词、记忆格式、检索脚本、上下文选择等，用于改善搜索质量，可自由编辑。
  - **评估组件**（Evaluation Components）：包括验证器、评分器等，定义解决方案有效性和任务指标。其编辑受限于真实世界反馈，只有在本地评估器的代理分数与真实评估器（oracle）分数出现分歧时才允许修复，避免无限制的奖励篡改。
- **关键技术细节**：
  - **统一行动空间**：模型可执行两类动作——提交对象级解决方案，或使用沙箱工具（如 `ls`、`cat`、`grep`、`replace`、`overwrite`）检查和修改可编辑 harness 文件。完成交互通过 `done` 命令。
  - **奖励设计**：不直接奖励 harness 编辑动作本身；只有最终提交解决方案并基于改进后的 harness 获得更高任务分数时，该轨迹才获得正面奖励。对于评估器编辑，奖励仅当代理分数与 oracle 分数不一致时给予，且与 mismatch set 匹配度成正比。
  - **相位级审查**（Phase-Level Review）：每个训练阶段包含多个 GRPO 轮次。阶段结束时，收集轨迹中产生正面 PoC 的引导编辑候选，经过过滤、排序、验证评分后，提交最佳稳定引导 harness。对于评估器编辑，则基于 mismatch set 和回归测试决定是否接受补丁。
  - **安全机制**：评估器编辑需满足：修复 mismatch 集中的更多案例，同时保留必要接口且不引起回归。这种设计使评估器修复是渐进式的，而非一次性。
- **算法流程**（文字描述）：
  1. 初始化策略 πθ、初始 harness h0、本地评估器 E_loc，可选真实评估器 E_real；若允许评估器进化，初始化 mismatch 集 M。
  2. 对于每个阶段 p：
     - 初始化候选池 C_p 为空。
     - 对于每个 GRPO 轮次 e：
       - 从 πθ 采样多轮轨迹（可能包含解决方案提交和 harness 编辑）。
       - 使用本地评估器评分提交的解决方案。
       - 分配轨迹奖励。
       - 将解析出的、具有正面 PoC 的引导编辑加入 C_p。
       - 使用 GRPO 更新 πθ。
     - 若允许评估器进化：查询真实评估器，将不一致案例加入 M；根据 M 和回归测试审查评估器编辑。
     - 审查引导编辑候选：过滤、排序、验证评分，提交最佳更新。
     - 更新持久 harness 为 h_{p+1}。
  3. 重复直至收敛或达到预设阶段数。

## 6. 主要结论与发现

- **HASE 使 Qwen3-8B 能够匹配或超越使用 GPT-OSS-120B + 外部 Claude Code 提议器的系统性能**：文本分类准确率 86.98%（vs. Meta-Harness 86.8%）；Alpha 因子挖掘在 RankIC 0.0302（vs. GPT-OSS-120B 0.0318），AER 14.35%（vs. 7.89%），IR 1.70（vs. 0.80），全面超越；圆填充达到 SOTA 2.635983；Heilbronn 达到 0.036082（vs. 冻结基线 0.0219 和已知 SOTA 0.0365）。
- **评估器修复是渐进且必要的**：圆填充和 Heilbronn 中，HASE 能检测代理分数与 oracle 分数的分歧，自动修复破损评估器，然后继续搜索找到优质方案。
- **引导 harness 的进化能提升搜索效率**：Alpha 因子挖掘中，模型从仅显示最近 3 个因子进化到按 IC 排序显示前 10 个因子，更有效地引导因子生成。
- **权重协同进化（RL 更新）是关键**：Heilbronn 消融表明，仅修复评估器而不更新模型权重，性能停滞在 0.0219，同步 RL 训练后可达 0.0361。

## 7. 优点

- **统一的行动空间设计**：将解决方案生成和 harness 编辑放在同一个多轮交互框架中，无需外部代码提议器，降低系统复杂度。
- **严格的评估器进化安全机制**：仅当本地评估器与真实评估器出现分歧时才允许编辑评估器，且通过 mismatch set 和回归测试确保补丁修复问题而不引入新问题，有效防止 reward hacking。
- **引导与评估器分离处理**：引导组件可自由优化，评估器编辑受 oracle 监督，体现差异化治理原则。
- **实证效果显著**：在多个迥异的任务上验证，8B 模型性能媲美甚至超越 120B 模型系统，展示出框架的泛化能力。
- **方法透明且可重复**：提供详细算法描述、演化代码示例（附录中给出多阶段 harness 代码）和审计流程。

## 8. 不足与局限

- **实验覆盖有限**：仅测试了 Qwen3-8B 一个骨干模型，且任务数量不多（4 个），未进行大规模缩放研究（如扩展到 70B、120B 或更多领域）。
- **保守的设计选择**：可编辑 harness 组件采用白名单方式；评估器编辑需要 oracle 分歧才能触发，可能拒绝某些有益但间接的修改；相位级审查仅提交接口验证通过的补丁，可能排除更创新的改动。
- **文本分类任务依赖与 Meta-Harness 相同的固定测试协议**：最终评测仅使用一次 greedy 推理，无测试时编辑，这可能限制了 HASE 在文本分类中的真实潜力（但保证了公平比较）。
- **Heilbronn 实验中初始评估器非常脆弱**：仅检查 NaN/无穷和点数，容易被利用；设计足够简单以显示修复过程，但真实场景的评估器可能更复杂，修复难度更大。
- **缺乏多轮交互的细粒度分析**：虽然展示了自我修正的例子（如文本分类中的工具编辑），但未量化多次尝试的成功率或平均修复轮数。
- **计算资源细节不完整**：未报告训练的具体耗时或 FLOPs，难以评估效率。
- **未探讨更复杂的场景**：如需要联合编辑多个评估组件、处理连续状态空间、或集成外部人类反馈等。

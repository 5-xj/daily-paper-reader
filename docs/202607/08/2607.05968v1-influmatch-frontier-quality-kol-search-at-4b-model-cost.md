## 摘要
将网络红人（KOL）与自由形式、多部分的泰语营销标准进行匹配，目前要么通过结构化档案的关键词搜索（忽略语义匹配），要么通过在每个候选上提示前沿大语言模型（准确但缓慢且昂贵）。我们提出InfluMatch，一种低成本的三阶段级联——检索→重排序→推理——完全由小型开放权重模型构建：稠密检索返回50个候选，一个4B点级重排序器根据单个“是”标记的对数概率对每个候选项评分并保留10个，以及一个4B推理器根据带有泰语理由的评分标准对候选名单进行评分。该级联设计为低成本：对过滤后的前10名进行推理，其token花费是对所有50名进行推理的一半，同时得分高出14分。在11个查询集（所有50个候选均标注）上，端到端与人类相关性标签对比，完整级联达到94.1%的P@5，而仅检索的基线接近随机；它与前沿模型Kimi-K2.6（91.8%）性能相当，同时输出token数量减少约35倍，并在单个A100上约20秒内处理一个50-KOL查询。值得注意的是，唯一有益的微调是成对微调：经过SimPO微调的重排序器与前沿基线的最佳选择准确率（78.0 EM）相匹配，而在点级每标准标签上微调推理器提高了离线分数却降低了端到端排名——我们将这种反转归因于绝对标注任务的设计——使得未微调的基础模型成为最强大的已部署推理器。结果是一个可部署、可解释的KOL搜索系统，其服务成本仅为前沿模型的一小部分。

## Abstract
Matching influencers (KOLs) to free-form, multi-part Thai marketing criteria is today served either by keyword search over structured profiles, which misses semantic fit, or by prompting frontier LLMs over every candidate, which is accurate but slow and expensive. We present InfluMatch, a low-cost three-stage cascade -- retrieval $\rightarrow$ rerank $\rightarrow$ reason -- built entirely from small open-weight models: dense retrieval returns 50 candidates, a 4B pointwise reranker scores each by the log-probability of a single Yes token and keeps 10, and a 4B reasoner grades the shortlist per criterion on a rubric with a Thai rationale. The cascade is designed for cost: reasoning over a filtered top-10 halves token spend versus reasoning over all 50 while scoring 14 points higher. End-to-end against human relevance labels on an 11-query set with all 50 candidates labeled, the full cascade reaches 94.1% P@5, versus a retrieval-only baseline near random; it matches the frontier model Kimi-K2.6 (91.8%) while emitting ${\sim}35\times$ fewer output tokens and serving a 50-KOL query in ${\sim}20$ s on one A100. Notably, the only fine-tuning that pays off is pairwise: a SimPO-tuned reranker matches the frontier baseline's best-pick accuracy (78.0 EM), whereas fine-tuning the reasoner on pointwise per-criterion labels improves offline scores yet degrades end-to-end ranking -- an inversion we trace to the design of the absolute labeling task -- leaving the untuned base model as the strongest deployed reasoner. The result is a deployable, explainable KOL search system at a small fraction of frontier serving cost.

- **意义**：填补了低成本工业级语义KOL匹配的空白，证明了小型开放权重模型（4B参数）通过精心设计的级联和相对偏好微调可以达到前沿质量，同时提供了关于监督信号设计的深刻见解（绝对评分 vs 相对判断）。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：**三级级联流水线（检索→重排→推理）**，将昂贵的推理过程限制在过滤后的小候选集上。全部使用4B参数的开源模型（Qwen3.5-4B作为基础，重排器使用SimPO微调，推理器使用未微调的Base）。
- **技术细节**：
  1. **检索（Retrieval）**：对每个营销标准嵌入查询，对KOL向量表进行稠密检索，每个KOL取与各标准的最小余弦距离之和作为分数，保留前50名候选。
  2. **重排（Rerank）**：使用一个4B点级重排器。输入营销标准和单个KOL的档案+摘要，输出**单个“Yes”token的对数概率**作为相关性分数，保留前10名候选。重排器通过**SimPO（Simple Preference Optimization）**在人类成对偏好数据上进行LoRA微调，目标函数为 \( \mathcal{L} = -\log \sigma(\beta (s_w - s_l) - \gamma) \)，其中 \( s = \log p(\text{Yes} | C, k) \)。
  3. **推理（Reason）**：使用一个4B推理器，对每个短名单KOL的每个标准，按{0,1,2}顺序评分，并生成泰语理由。最终排序分数为各标准分数之和（等权和）。**部署使用的推理器是未微调的Base模型**（因为微调后的版本在端到端上反而不如Base）。
- **为什么级联有效**：重排器擅长相对过滤（但精度有限），推理器擅长精确评分（但分辨率粗，易导致大池子中的平局和近平局）。先过滤再推理，各司其职，既便宜又准确。

## 6. 论文的主要结论与发现

1. **级联优于直接推理**：SimPO重排 + Base推理在top-10上达到**94.1% P@5**，而直接对所有50推理仅80.0%，且token花费更低。
2. **成对SimPO重排器匹配前沿**：4B SimPO重排器在最佳选择准确率上（78.0 EM）与Kimi-K2.6持平，而未经训练的Base重排器仅63.4%。
3. **点级微调导致离线-线上反转**：SFT+GRPO推理器离线wf1最高（59.0），但在端到端中性能最差（85.9% P@5 vs Base的94.1%）。原因：
   - 绝对评分任务标签质量差：证据审计显示仅15.9%的“匹配”标签有档案证据支持。
   - 标注界面分组导致标注内相关。
   - 点级目标与端到端整体相关性不完全一致。
4. **限制来源于标签，而非模型容量**：在证据清晰的子集上，4B推理器wf1可升至73.4-80.6，表明模型本身能力充分。
5. **文本重排优于添加图像**：视觉重排仅带来+1.8 P@5，但增加19%输入token和3.3倍延迟，不划算。

## 7. 优点

- **成本效益极高**：使用4B模型实现94%+ P@5，输出token仅3k，匹配前沿模型但计算成本低若干数量级。
- **可解释性**：输出每标准评分和泰语理由，便于营销人员审计。
- **设计简洁有效**：级联架构（检索→重排→推理）不仅节省成本，还通过“分而治之”提升了排名质量。
- **监督信号洞察深刻**：通过证据审计揭示了绝对标注任务的内在噪声，并证明相对判断（成对偏好）才是该场景下有效的反馈信号。
- **部署友好**：可在单张A100上20秒内响应，适合生产环境。

## 8. 不足与局限

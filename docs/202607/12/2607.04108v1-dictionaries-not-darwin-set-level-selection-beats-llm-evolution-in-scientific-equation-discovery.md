## 摘要
大型语言模型越来越多地被用作科学发现的进化引擎：生成候选、选择胜出者、将其作为父代反馈并重复。我们审计了这一循环是否真的在科学方程发现（一个有限样本导致结构欠定且插值容易的场景）中增强了发现能力。在匹配的LLM调用预算下，父代条件化进化与独立的新鲜采样无法区分：中位数OOD NMSE为0.045对比0.049，有指导的多父代交叉更差，最终成功由初始提议质量预测，且多次迭代方案未能增加已解决问题。操作上，该循环简化为其产物：候选术语的词典。我们将这一诊断转化为PTB-Search，一种单代组件化科学发现方法。PTB-Search一次性采样独立的LLM提议，将可重用术语提取到每个问题的词典中，并执行仅基于训练数据的集合级稀疏选择（使用最小二乘系数）。其核心原则是，欠定数据识别的是术语集合的联合行为，而非可靠的单个术语贡献。在相同的词典和零额外LLM调用下，集合级选择器解决了717个单元中的165-169个，而单项缩减仅解决74-78个。在官方的239问题LLM-SRBench划分上，PTB-Search使用Llama-3.1-8B达到73.2%的Acc0.1，使用单种子DeepSeek-V4锚点达到77.0%，而最佳报告基线为49.2%，且使用的标准化调用预算仅为十分之一。程序域压力测试给出了边界范围：生成次数仍不可靠，而保留外部状态在更困难的非线性空间中可能有帮助。综合这些结果，LLM最好被理解为材料供应商；发现由对可重用组件的外部集合级选择承载。

## Abstract
Large language models are increasingly used as evolutionary engines for scientific discovery: generate candidates, select winners, feed them back as parents, and repeat. We audit whether this loop actually compounds discovery in scientific equation discovery, a setting where finite samples make structure underdetermined and interpolation easy. Under matched LLM-call budgets, parent-conditioned evolution is indistinguishable from fresh independent sampling: median OOD NMSE is 0.045 vs. 0.049, instructed multi-parent crossover is worse, final success is predicted by initial proposal quality, and multiple iteration schemes fail to add solved problems. Operationally, the loop reduces to what it produces: a dictionary of candidate terms. We turn that diagnosis into PTB-Search, a one-generation method for componentized scientific discovery. PTB-Search samples independent LLM proposals once, extracts reusable terms into a per-problem dictionary, and performs train-only set-level sparse selection with least-squares coefficients. Its central principle is that underdetermined data identifies the joint behavior of term sets, not reliable per-term credit. On identical dictionaries and zero additional LLM calls, set-level selectors solve 165--169 of 717 cells, while single-term reductions solve only 74--78. On the official 239-problem LLM-SRBench split, PTB-Search reaches 73.2% Acc0.1 with Llama-3.1-8B and 77.0% with a single-seed DeepSeek-V4 anchor, versus 49.2% for the best reported baseline, using one tenth of the standardized call budget. A program-domain stress test gives a scoped boundary: generation count remains unreliable, while retained external state can help in harder non-linear spaces. Across these results, LLMs are best understood as material suppliers; discovery is carried by external set-level selection over reusable components.

## 2. 方法论
### 核心思想
- **材料而非轨迹**：LLM 提议的价值在于哪些术语存在于提议间，而非公式如何被重写。因此，设计应：一次采样独立提议 → 提取术语 → 构建字典 → 执行仅基于训练数据的集合级稀疏选择。
- **集合级可识别性**：欠定数据只能识别术语集合的联合行为，而非单个术语的贡献。选择信号必须在集合级计算，任何降为单术语统计的方法都会丢失信号。

### 关键技术细节
1. **词典构建**：从 n=100 个独立 LLM 提议中解析出加性项，标准化后与一组通用基函数（如多项式、指数、对数等）合并，形成每个问题的候选字典 D = {φ₁,…,φₚ}。
2. **假设类**：稀疏线性组合 ŷ = β₀ + Σⱼᵤ βᵠ(x)，支持集大小 |S| ≤ 6；系数用最小二乘拟合。
3. **选择算法**：束搜索（beam search）在字典上评分支持集，评分指标为内部训练分裂的 NMSE（采用最小二乘拟合），并添加稳定性生成的支持候选、成对协同感知评分、以及一个训练侧守卫折用于打破统计打平。
4. **停止条件**：无代际迭代，一次完成。

### 设计强制由消融实验支撑
- 单个提议 → 字典 → 性能崩溃（开发集中位数 OOD NMSE 从 4.7×10⁻⁹ 降至 2.2×10⁻²）。
- 密集拟合（无子集选择）统一更差。
- 跨问题族移植术语完全失败。
- 增加代际迭代不可靠（无增益甚至破坏性）。
- 支持大小上限 ≤6 由复杂度预算决定（放宽只会增长公式长度而非准确度）。

## 6. 主要结论
1. **进化循环无效**：在方程发现中，父代条件化变异与新鲜采样无显著差异（中位数 OOD NMSE 0.045 vs. 0.049）；显式交叉更差；初始提议质量决定最终成功（AUC 0.959）；三类迭代方案均不能增加已解决问题数。
2. **PTB-Search 显著优于基线**：使用 Llama-3.1-8B 达到 73.2% Acc0.1，DeepSeek-V4 锚点 77.0%，而最佳基线（LLM-SR）仅 49.2%，且预算仅为十分之一。
3. **选择必须位于集合级**：在相同字典和零额外 LLM 调用下，三个集合级选择器解决 165–169 个单元，两个单术语选择器仅解决 74–78 个；跨家族差距极端显著（p < 10⁻⁷³）。
4. **代际迭代无累积价值**：三类迭代方案在方程和程序域均未展示超过第 0 代的增益；程序域困难网格中，增益来自外部种群/交叉探针，而非父代条件化生成次数。
5. **故障分解**：剩余误差大部分由提议器限制（缺乏术语）或选择器限制导致，组合限制极低（≤5%）；复杂度守卫显示非线性层的增益被公式长度膨胀抵消。
6. **跨提议器稳定性**：弱模型（Llama）与强模型（DeepSeek）在相同选择器下仅差 4 个百分点，说明选择器是负载核心。

## 7. 优点
- **方法简洁高效**：单代设计，无迭代，计算开销低，但性能大幅超越复杂循环方法。
- **理论扎实**：从欠定数据可识别性出发，严格论证集合级选择的必要性，并提供直接实验验证。

## 8. 不足与局限
- **应用范围限定**：论文最强结论针对组件化科学表达式发现（如加性项字典），不直接适用于任意程序合成或非组件化空间。
- **符号准确率受限**：PTB-Search 原生输出稀疏加性模型，与紧凑封闭形式符号等价性差，因此符号准确率报告仅为诊断性数值，非核心指标。

- **未覆盖所有可能系统**：结论主要针对当前主流的简单进化循环，不排除未来更复杂机制的可能性。

## 摘要
科学创意生成（Scientific Idea Generation, SIG）是人工智能驱动的自主科研的关键环节，但现有方法往往受限于静态的“检索-生成”范式，导致产生的想法趋于同质化且缺乏足够的发散性。本文提出 FlowPIE，这是一种紧密耦合的检索-生成框架，将文献探索与创意生成视为协同进化的过程。FlowPIE 通过受 GFlowNets 启发的流引导蒙特卡洛树搜索（MCTS）扩展文献探索路径，并以由大型语言模型（LLM）支持的生成性奖励模型（Generative Reward Model, GRM）评估的当前创意质量作为监督信号，指导自适应检索并构建一个多样化、高质量的初始创意群体。在此基础上，FlowPIE 将创意生成建模为测试时的创意进化过程，结合隔离岛范式和基于 GRM 的适应度计算，执行选择、交叉和变异操作，以融合跨领域知识。该方法有效缓解了对参数化知识和静态文献的过度依赖所导致的信息茧房问题。大量实验表明，与强大的基于 LLM 或智能体的框架相比，FlowPIE 在新颖性、可行性与多样性方面持续产出更优的创意，同时可在测试阶段实现奖励扩展。

## Abstract
Scientific idea generation (SIG) is critical to AI-driven autonomous research, yet existing approaches are often constrained by a static retrieval-then-generation paradigm, leading to homogeneous and insufficiently divergent ideas. In this work, we propose FlowPIE, a tightly coupled retrieval-generation framework that treats literature exploration and idea generation as a co-evolving process. FlowPIE expands literature trajectories via a flow-guided Monte Carlo Tree Search (MCTS) inspired by GFlowNets, using the quality of current ideas assessed by an LLM-based generative reward model (GRM) as a supervised signal to guide adaptive retrieval and construct a diverse, high-quality initial population. Based on this population, FlowPIE models idea generation as a test-time idea evolution process, applying selection, crossover, and mutation with the isolation island paradigm and GRM-based fitness computation to incorporate cross-domain knowledge. It effectively mitigates the information cocoons arising from over-reliance on parametric knowledge and static literature. Extensive evaluations demonstrate that FlowPIE consistently produces ideas with higher novelty, feasibility and diversity compared to strong LLM-based and agent-based frameworks, while enabling reward scaling during test time.

## 二、方法论：FlowPIE 框架

### 1. 核心思想

- 把“文献探索（retrieval）”与“创意生成（generation）”统一为一个**协同进化过程（co-evolution process）**；
- 利用 **流引导蒙特卡洛树搜索（Flow-guided MCTS）** 动态探索专利文献图谱；
- 结合基于 LLM 的生成性奖励模型（Generative Reward Model, GRM）对创意质量进行测试时反馈；
- 以进化算法（Evolutionary Algorithm）形式进行创意迭代，包括选择、交叉、变异；
- 引入 **隔离岛（Isolation Island）** 策略，实现跨领域、多样化的知识融合。

### 2. 技术细节与流程（文字化说明）

1. **数据建模**：  
   - 使用来自美国专利局（USPTO）的**专利文献**构建异构图 \( G = (V, E) \)，节点为专利，边基于引用关系、核心技术特征重叠或语义相似度建立。  

2. **流引导 MCTS (Flow-guided MCTS)**：  
   - 以用户查询为根节点，执行基于下式的探索决策：
     \[
     UCB(s'|s) = Q(s'|s) + c · P_f(s'|s)\sqrt{N(s)} / (1 + N(s'|s))
     \]
     其中：
     - \( Q(s'|s) \)：边的预期回报；
     - \( P_f(s'|s) \)：流概率，反映文献的重要度；
     - \( c \)：探索系数；
     - \( N(\cdot) \)：访问次数。
   - LLM基于路径生成创意，GRM对创意打分（奖励 \(R\) ），再反向传播更新流：
     \[
     P_f ← (1−α)P_f + α\tilde{R}_t
     \]
     实现“高质量创意 → 高权重文献”的动态调整。

3. **创意进化阶段（Test-Time Idea Evolution）**：
   - **选择（Selection）**：基于GRM得分进行锦标赛式筛选；
   - **交叉（Crossover）**：LLM融合两组高分创意与关联文献，重组核心技术点；  
   - **变异（Mutation）配合隔离岛策略**：从拓扑上远离当前领域的文献子图中采样，引入跨域信息；
   - **适应度评估（Fitness Evaluation）**：LLM-GRM综合评估新创意的可行性与新颖性；
   - **迭代终止条件**：当奖励方差低于阈值或达到最大迭代次数。

4. **整体算法组成**
   - 算法 1：基于 MCTS 的初始创意群体构建；
   - 算法 2：测试时创意进化；
   - 均采用深度衰减奖励与移动平均流概率更新机制。

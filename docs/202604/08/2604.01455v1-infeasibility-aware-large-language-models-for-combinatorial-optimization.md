## 摘要
大型语言模型（LLMs）正日益被用于求解 NP 难的组合优化问题，但现有的大多数方法主要关注生成可行实例的解，而未显式处理不可行性的检测问题。我们提出了一种具备不可行性感知能力的框架，结合了可证明的数据集构建、监督微调以及基于 LLM 的下游搜索。针对 minor-embedding 问题，我们提出了一种新的数学规划形式化方法，并引入可证明的零阶段不可行性筛选，从而实现了带有结构化证书的可行实例与可认证不可行实例的可扩展训练集构建。利用通过该精确优化流程生成的训练数据，我们展示了一个拥有 80 亿参数的 LLM 可以通过微调实现解生成与不可行性检测的联合执行。我们进一步将 LLM 的输出作为下游局部搜索的初始解，从而提供了一种在 LLM 输出不完美时仍能加速优化过程的实用方法。实验结果表明，我们的微调模型相比 GPT-5.2 将总体精度提升了多达 30%，同时基于 LLM 引导的初始解在下游局部搜索中可实现最高 2 倍的加速效果。

## Abstract
Large language models (LLMs) are increasingly explored for NP-hard combinatorial optimization problems, but most existing methods emphasize feasible-instance solution generation and do not explicitly address infeasibility detection. We propose an infeasibility-aware framework that combines certifiable dataset construction, supervised fine-tuning, and LLM-assisted downstream search. For the minor-embedding problem, we introduce a new mathematical programming formulation together with provable zero-phase infeasibility screening, which enables scalable construction of training instances labeled either as feasible with structured certificates or as certifiably infeasible. Using training data generated through this exact optimization pipeline, we show that an 8B-parameter LLM can be fine-tuned to jointly perform solution generation and infeasibility detection. We further utilize LLM outputs as warm starts for downstream local search, providing a practical way to accelerate optimization even when the LLM outputs are imperfect. Experiments show that our fine-tuned model improves overall accuracy by up to 30\% over GPT-5.2; meanwhile LLM-guided warm starts provide up to $2\times$ speedup compared with starting from scratch in downstream local search.

## 二、方法论与技术方案

### 1. 框架结构

框架整体分为三个阶段：

1. **数据构建（Certifiable Dataset Construction）**  
   - 通过**数学规划**精确生成带有“可行”或“不可行”标签的训练数据；
   - 每个实例均附带可验证的“结构化证书”（solution certificate 或 infeasibility proof）。

2. **监督微调（Supervised Fine-Tuning, SFT）**  
   - 使用 LoRA 参数高效微调技术（在 Llama-3.1 8B 模型上）；
   - 模型学习以统一的输出格式同时：
     - 生成可行解；
     - 输出不可行声明（"No" verdict）。

3. **推理与修复（LLM-assisted Downstream Search）**  
   - 模型输出用于初始化局部搜索算法（Feasibility Jump, FJ）；
   - LLM 生成结果若不可行，可被 FJ 迅速“修复”，实现快速逼近最优可行解。

### 2. 数学规划与零阶段不可行检测

#### (1) Minor-embedding 数学形式化

- 定义：
  - 目标：将一个“问题图” P 嵌入到“硬件图” G；
  - 约束：
    1. 每个问题顶点映射到一个连通子集（chain）；
    2. 不同顶点的 chain 互不相交；
    3. 每个问题边在 G 中至少对应一个连接 chain 的硬件边。
  
- 使用的整数规划表达式：
  \[
  \min \sum_{i,C}s(C)x_{iC}
  \]
  约束：
  - 每个问题顶点唯一对应一个 chain；
  - 不同 chain 不重叠；
  - 相邻顶点映射的 chain 必须相邻。

- 该问题在“显式 chain 枚举形式”下仍 NP-完全，论文附带完整证明。

#### (2) 零阶段不可行筛选（Zero-phase Screening）

- 作者提出必要条件（命题 1）：
  \[
  \deg_P(i) \le L(\Delta_H -2)+2,\quad
  \sum_i s_i^{min} \le |V_H|, \quad
  |E_P|+\sum_i (s_i^{min}-1) \le |E_H|
  \]
  任意违反即“可判定不可行”，**无需求解器调用**；
  - 该方法在数据生成阶段大幅减少计算成本。

### 3. 模型训练策略

- **目标函数**：标准自回归 SFT 损失；
- **输出格式**：  
  用统一、可验证的输出模版，包括两类输出：
  - `yes + 具体结构化解`
  - `no + infeasible declaration`
- **策略创新**：  
  不使用“思维链”推理，而采用紧凑一体化输出，减少 token 消耗，提升速度与一致性。

### 4. 推理机制

- **Best-of-N Sampling**：采样 N 次输出，用验证模块选择可行证书；
- **Feasibility Jump (FJ)**：若输出不完全可行，则作为局部搜索的初始解进行迭代修补，加速整个优化过程。

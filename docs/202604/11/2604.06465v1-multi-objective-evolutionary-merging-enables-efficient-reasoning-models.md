## 摘要
推理模型通过利用长链式思维在解决复杂问题方面展现出卓越的能力。然而，这种更为深思熟虑的推理在推理阶段带来了显著的计算开销。Long-to-Short（L2S）推理问题旨在使用更少的token保持高准确率，但当前无需训练的模型融合方法依赖于标量化的、固定超参数的算术手段，这些方法高度脆弱且往往导致次优折中。为了解决这一问题，我们提出了Evo-L2S，一个将L2S推理表述为多目标优化挑战的新框架。通过演化式模型融合，Evo-L2S显式优化准确性与输出长度之间的权衡，从而生成一组稳健的帕累托前沿融合模型。为了使该搜索在大规模语言模型上计算可行，我们提出了一种基于熵的子集采样技术，大幅降低了适应度估计的开销。在1.5B、7B和14B参数规模下的六个数学推理基准上进行的全面实验表明，Evo-L2S可以在保持甚至提升原始推理模型问题求解准确率的同时，将生成的推理链长度减少超过50%。

## Abstract
Reasoning models have demonstrated remarkable capabilities in solving complex problems by leveraging long chains of thought. However, this more deliberate reasoning comes with substantial computational overhead at inference time. The Long-to-Short (L2S) reasoning problem seeks to maintain high accuracy using fewer tokens, but current training-free model merging approaches rely on scalarized, fixed-hyperparameter arithmetic methods that are highly brittle and force suboptimal compromises. To address this gap, we introduce Evo-L2S, a novel framework that formulates L2S reasoning as a multi-objective optimization challenge. By leveraging evolutionary model merging, Evo-L2S explicitly optimizes the trade-off between accuracy and output length to produce a robust Pareto front of merged models. To make this search computationally tractable for large language models, we propose an entropy-based subset sampling technique that drastically reduces the overhead of fitness estimation. Comprehensive experiments across 1.5B, 7B, and 14B parameter scales on six mathematical reasoning benchmarks demonstrate that Evo-L2S can reduce the length of generated reasoning traces by over 50% while preserving, or even improving, the problem-solving accuracy of the original reasoning models.

## 一、研究问题与背景

- **核心问题**：当前大型语言模型（LLMs）在复杂推理任务中表现突出，但长链式推理（Chain-of-Thought，CoT）导致推理阶段的计算成本极高。  
- **研究动机**：  
  - 复杂的“System 2”类型深度推理虽能提升准确率，但伴随大量冗余思考与“过度推理”(overthinking) 问题。  
  - 实际应用中需同时保持高推理准确率和低计算开销。  
- **中心任务**：Long-to-Short (L2S) 推理问题要求在不重新训练模型的情况下，通过调整模型参数实现推理长度显著减少，同时维持高精度。  
- **研究缺口**：以往“任务算术 (Task Arithmetic)”等训练自由模型融合方法依赖固定超参数，难以兼顾准确率与推理长度的非线性权衡，且高度不稳定。

## 二、方法论：Evo-L2S 框架

### 核心思想
- 将 **L2S 推理问题视为多目标优化问题**。
- 通过 **进化式模型融合 (Evolutionary Model Merging)** 自动优化“准确率”和“输出长度”两个相互冲突的目标。
- 不再依赖人为设定的单一权重或超参数，而是利用多目标进化算法探索帕累托前沿（Pareto Front）上的最佳权衡点。

### 技术细节
1. **模型融合公式**  
   - 从两个共享预训练初始化的模型出发：  
     - System 1（快速思考）：Qwen2.5-Math 系列；  
     - System 2（缓慢思考）：DeepSeek-R1-Distill-Qwen 系列。  
   - 定义参数差向量：  
     \[
     \tau = \theta_{S1} - \theta_{S2}
     \]
   - 对应的融合形式（任务算术）：  
     \[
     \theta_M = (1 - \lambda)\theta_{S2} + \lambda \theta_{S1}
     \]
     其中λ为进化搜索中的基因型变量。
2. **进化算法机制**  
   - 使用 NSGA-II 多目标进化算法：
     - 目标函数：同时最大化准确率、最小化生成长度；
     - 选择策略：基于非支配排序与拥挤距离保持多样化。
   - 每代评估多个候选融合模型，并挑选非支配解形成近似帕累托前沿。
3. **熵驱动的子集采样**（关键创新）
   - 为降低评估成本，提出**基于信息熵的高效样本筛选**：
     - 计算每个问题在不同融合模型上的正确率分布；
     - 按 Bernoulli 熵选取最具区分性的样本（即各候选模型正确率接近0.5的问题）。
   - 实验验证此方法能在仅50题的子集上复现全测试集排序结果（Spearmanρ≈1）。

## 三、实验设计

### 数据与基准
- 六个标准数学推理 benchmark：
  - GSM8K、MATH500、Minerva-Math、OlympiadBench、College-Math、AIME24。
- 各基准涵盖基础算数、复杂逻辑、多步推理到奥数题级别难度。

### 模型规模
- 实验在三个参数规模下进行：
  - **1.5B、7B、14B 参数量级**。
- 每个规模合并同架构的 Qwen2.5-Math 与 DeepSeek-R1-Distill-Qwen 模型。

### 对比方法
- **训练自由（merging）类基线：**
  - 平均融合 (Average Merging)
  - Task Arithmetic (TA)
  - TIES-Merging
  - Activation-guided Consensus Merging (ACM)
- **其他优化策略：**
  - 单目标演化融合（仅优化准确率或长度）
  - Evo-L2S 多目标演化（提出方法）

### 解码与评估配置
- 使用 vLLM 框架，bfloat16 精度，greedy decoding（temp=0.0, top-p=1.0）。
- System 1 few-shot，System 2 zero-shot。
- 统一生成参数上限：8192 vs. 10240 tokens。

## 六、主要结论与发现

- Evo-L2S 在三个参数规模上均显示出显著优势：
  - 平均**推理长度减少 50%+**；
  - 几乎不损失准确率，部分基准上甚至超越原始 System 2 模型。

  - 14B规模：最长可减少77%，仍维持接近基线精度。
- 熵采样显著提升搜索效率——在仅50样本子集上维持近乎完美的排名一致性。

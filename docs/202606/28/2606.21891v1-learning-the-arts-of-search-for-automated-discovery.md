## 摘要
科学发现可以表述为在假设和实验空间上的迭代搜索过程。当代方法使用MCTS等启发式算法来导航这一空间。这些算法将假设的优劣与其实验执行的质量混为一谈。因此，一个有前景但初步执行的假设被排在了一个执行更精细但平庸的假设之后。此外，先前的方法会随着搜索的进展修剪搜索日志，因为累积的历史记录超出了上下文窗口。我们提出了用于树搜索的代理推理（ARTS），其中我们部署了一个推理语言模型来导航这一空间。该模型检查先前的执行日志，诊断早期的失败是由于错误实现还是不良假设导致的，并选择接下来要基于哪个假设进行构建。为了缓解上下文长度带来的挑战，ARTS使用测试时训练将搜索树的知识灌输到模型权重中。在来自MLGym和MLEBench的22个任务上，我们展示了ARTS优于领先算法，标准化得分相对提升超过15.3%。通过测试时训练，我们展示了Qwen3-4B智能体可以在推理成本降低高达5倍的情况下，与Gemini-3 Pro和GPT o3推理等闭源前沿模型媲美性能。我们进一步观察到，在部分可观测的强化学习任务上，测试时训练后的Qwen3-4B科学家通过重新发现人类最佳循环记忆解决方案（启发式方法会修剪掉该方案），超越了使用o3科学家的ARTS。

## Abstract
Scientific discovery can be formulated as an iterative search process over the space of hypotheses and experiments. Contemporary methods navigate this space using heuristics such as MCTS. These algorithms conflate the merit of a hypothesis with the quality of its experimental execution. A promising hypothesis with preliminary execution is therefore ranked below a modest hypothesis whose execution is refined. Moreover, prior methods prune the search logs as the search progresses because the accumulated history outgrows the context window. We propose Agentic Reasoning for Tree Search (ARTS), where we deploy a reasoning language model to navigate this space. The model inspects prior execution logs, diagnoses whether earlier failures arose from faulty implementations or bad hypotheses, and selects the hypothesis to build on next. To mitigate challenges with context length, ARTS uses test-time training to instill the knowledge of search tree in the model weights. Across 22 tasks from MLGym and MLEBench, we show that ARTS outperforms leading algorithms, with over 15.3% relative improvement in the normalized score. With test-time training we show that a Qwen3-4B agent can match performance with closed-source frontier models like Gemini-3 Pro and GPT o3-reasoning with upto 5x lower inference cost. We further observe that on partially observable RL tasks, the test-time trained Qwen3-4B scientist surpasses ARTS with the o3 scientist by rediscovering the human-best recurrent-memory solution that heuristic methods prune away.

## 1. 核心问题与整体含义（研究动机和背景）

科学发现本质上是在假设和实验空间中的迭代搜索过程。现有自动化方法（如MCTS、线性搜索、进化搜索）使用基于分数的启发式规则来选择下一个实验节点，但存在两个根本缺陷：
- **混淆假设优劣与执行质量**：一个优秀的假设可能因代码bug、训练不足等实现问题而得分低，而一个平庸假设却因实现完善而得分高。启发式方法倾向于优先扩展得分高的节点，从而过早放弃有潜力的假设。
- **历史信息截断**：随搜索树增长，累积的执行日志超出模型上下文窗口，现有方法要么总结要么剪枝，导致信息丢失，影响后续决策。

论文提出**ARTS（Agentic Reasoning for Tree Search）**，用**大型推理模型（LRM）**作为“科学家”来驱动搜索，替代启发式节点选择。进一步提出**ARTS\***，通过**测试时训练（Test-Time Training）**将搜索树知识编码到模型权重中，解决上下文长度限制问题。

## 2. 方法论

### 核心思想
将搜索过程分为两个角色：
- **Scientist（科学家）**：使用推理模型（如o3）分析历史节点，诊断失败原因（想法错误 vs 实现错误），选择待扩展的父节点并生成新假设。
- **Executor（执行者）**：使用编码模型（如Gemini 3 Flash）根据科学家指令实现代码并运行实验。

### 关键技术细节
- **节点检查（Inspect）**：科学家可以查看任一节点的代码、训练曲线、执行日志，从而识别低分原因。
- **基线审计（Audit）**：搜索初期保留若干轮，仅允许科学家阅读数据、基线代码、评估脚本，产生任务结构分析（如度量特性、信息损失最大的步骤）。
- **假设采样（Verbalized Sampling）**：科学家枚举K个候选假设并分配概率，由外部采样器随机选择一个，避免模式坍塌。
- **记忆模块（Memory）**：每次扩展后写入一条简短洞察，供科学家后续读取，缓解上下文膨胀。
- **测试时训练（ARTS\*）**：当搜索树超出上下文限制时，使用LoRA对科学家模型进行GRPO（Group Relative Policy Optimization）在线微调，以当前搜索树为训练数据，以百分位奖励（percentile reward）优化父节点选择和假设生成策略。奖励函数：-0.5（无效）、-0.2（低于基线）、0（基线到70百分位）、1（≥70百分位）。单步奖励（one-step reward）比多步奖励更利于信用分配。

### 算法流程（文字说明）
1. 初始化搜索树，包含基线代码和分数。
2. 进行若干轮审计（仅阅读，不扩展）。
3. 循环直到时间预算耗尽：
   - 科学家检查当前树，获取日志。
   - 科学家根据日志、记忆输出K个候选假设及其概率，并选择父节点。
   - 外部采样器从K个假设中抽取一个。
   - 执行者实现该假设并验证，获得分数。
   - 将新节点加入树。
4. ARTS\*：每隔若干步，用当前树的节点进行GRPO训练，更新LoRA适配器，然后继续搜索。

## 6. 主要结论与发现

1. **ARTS优于现有方法**：在22个任务中的16个取得最佳，归一化分数提升15.3%，IQM 0.93 vs 0.87，最优性差距更小。
2. **失效归因避免了过早放弃有希望的假设**：通过检查日志区分“想法错误”与“实现错误”，保留有潜力的方向（如HMS、APTOS、Vesuvius）。
3. **假设多样性更高**：ARTS覆盖更多策略轴（4.43 vs 4.05），熵更高（1.73 vs 1.35），45%的扩展开辟全新假设（vs AIRA 3%、MLEvolve 0%），假设间TF-IDF距离更大。
4. **假设质量更高**：每次假设基于具体失败日志提出，而非随机变异，因此更高质量，预算利用更高效。
5. **测试时训练有效性**：Qwen3-4B经TTT后，在MLGym上平均归一化分数从0.72提升至1.01，在语言建模和MetaMaze上超越人类最佳，且匹配o3性能，推理成本降低5倍。
6. **TTT帮助克服上下文限制**：在MetaMaze任务中，TTT使模型重新考虑并优化了之前因上下文漂移而丢失的LSTM循环记忆方案，达到53.0奖励，超过人类最佳52.5。

## 7. 优点

- **创新方法论**：将推理模型（LRM）用于树搜索节点选择，替代启发式，实现基于原因的智能搜索。
- **有效解耦假设与执行**：通过节点检查和失效分类，避免评价偏差。
- **测试时训练巧妙解决上下文溢出**：将搜索树知识内化到模型权重，既保留历史信息又保持推理可靠性。
- **模块化设计**：科学家和执行者可独立升级，兼容不同能力的模型。
- **实验充分且公平**：相同预算、相同执行者、相同容器，多次运行，统计指标可靠。
- **开源代码和轨迹**：促进可复现性。

## 8. 不足与局限

- **依赖模型质量**：科学家（如o3）和执⾏者（如Gemini 3 Flash）的质量直接影响效果；若使用较弱模型，可能无法正确诊断失效原因。
- **计算成本偏高**：TTT需要额外GPU和显存，虽然推理成本降低，但训练开销增加。
- **仅验证ML研究任务**：未在更广泛科学领域（如化学、生物）进行测试，通用性尚待证明。
- **实验任务数量有限**：22个任务虽然多样，但相比实际科研场景仍显不足，尤其是强化学习任务仅有4个。
- **记忆模块容量有限**：每条记忆仅一句话，长期搜索仍可能丢失细节。
- **伦理风险**：强化学搜索能力可能被滥用（如自动化攻击性研究），需保持人类监督。
- **启发式方法基线可能不是最优**：AIRA和MLEvolve的实现版本未必完全代表该类方法的全部潜力，但公平比较下结论仍然稳健。

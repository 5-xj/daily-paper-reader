## 摘要
将不同聚类场景下的图像聚类统一起来仍然具有挑战性，因为任务之间存在根本性差异。我们提出了一种指南驱动的图像聚类代理，这是第一个通过文本指南弥合这些差距的通用框架。为了在不进行特定任务训练的情况下纳入复杂指南，我们提出了生成式概念代理建模，通过概念代理提取生成指南感知嵌入。对于需要自动发现簇的场景，我们引入了基于最小生成树的LLM遍历，有选择地应用LLM推理进行复杂语义判断。我们的方法适用于各种不同的聚类场景，从一般到细粒度分类，从全局到局部标准，以及从平衡到长尾分布。我们的框架在各种聚类任务中始终优于专门方法。

## Abstract
Unifying image clustering across different clustering scenarios remains challenging due to fundamental gaps among tasks. We introduce a Guideline-Driven Image Clustering Agent, the first universal framework that bridges these gaps through textual guidelines. To incorporate complex guidelines without task-specific training, we propose Generative Concept Proxy Modeling, which generates guideline-aware embeddings via concept proxy extraction. For scenarios requiring automatic cluster discovery, we introduce LLM Traversal based on Minimum Spanning Tree that selectively applies LLM reasoning for complex semantic judgments. Our method generalizes across diverse clustering scenarios spanning from general to fine-grained categorization, from global to local criteria, and from balanced to long-tail distributions. Our framework consistently outperforms specialized methods across diverse clustering tasks.

### 2. 论文提出的方法论

*   **核心思想**：提出一个名为**Guideline-Driven Image Clustering Agent**的混合代理框架。该框架分两阶段工作：
    1.  **第一阶段（高效嵌入）**：将用户提供的复杂文本指南转化为高质量的、指南感知的图像嵌入向量。
    2.  **第二阶段（智能推理）**：基于嵌入进行初步聚类，然后仅对需要复杂语义理解的合并决策，引入大语言模型（LLM）进行判断，从而在性能和效率之间取得平衡。

*   **关键技术细节**：

    1.  **生成式概念代理建模 (Generative Concept Proxy Modeling, GCPM)**
        *   **目标**：解决直接使用多模态指令感知嵌入模型时，关键属性可能被淹没或不相关视觉特征占主导的问题。
        *   **流程**：
            1.  利用多模态大语言模型 (MLLM) 根据文本指南A对图像 `x_i` 生成**概念代理描述** `c_i`。这个步骤强制性地将指南中指定的属性显式地提取为文本形式，实现属性解耦。
            2.  使用指令感知嵌入模型将概念代理描述编码成嵌入向量 `h_i`。
        *   **优点**：通过解耦属性，生成的嵌入能更好地反映指南意图，并支持标准聚类算法（如K-Means）。

    2.  **基于最小生成树的LLM遍历 (MST-based LLM Traversal)**
        *   **目标**：在自动发现簇的场景下，克服HDBSCAN等算法难以合并相似簇的问题，同时又避免对所有簇对进行昂贵的LLM比较。
        *   **流程**：
            1.  **初始聚类**：使用HDBSCAN对GCPM生成的嵌入进行第一轮聚类，产生大量微小但同质的簇。
            2.  **MST构建**：基于Ward距离计算所有簇之间的相似度矩阵，构建一个最小生成树。这能确保评估时优先考虑最可能合并的簇对。
            3.  **选择性LLM合并**：在MST上，按距离从小到大的顺序，依次调用LLM判断两个簇是否应该根据指南合并（`p = fmerge(G, Ci, Cj)`）。决策结果被缓存以避免重复查询。
        *   **算法流程**：算法以HDBSCAN的输出簇和孤立点为起点，迭代地构建MST，遍历边，并调用LLM做出合并决策，直到没有更多合并发生或因其他条件终止。
        *   **理论优势**：论文理论分析证明，其LLM调用次数的期望复杂度为 `O(M log M)`，远优于朴素方法的 `O(M²)`。

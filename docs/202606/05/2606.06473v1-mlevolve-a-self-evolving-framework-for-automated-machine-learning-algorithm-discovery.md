## 摘要
大型语言模型（LLM）智能体越来越多地被应用于长期任务，例如科学发现和机器学习工程（MLE），其中持续自演化成为关键能力。然而，现有的MLE智能体存在分支间信息隔离、无记忆搜索以及缺乏层级控制等问题，这些问题共同阻碍了长期优化。我们提出了MLEvolve，一个基于LLM的自演化多智能体框架，用于端到端的机器学习算法发现。通过将树搜索扩展为渐进式MCGS，MLEvolve通过基于图的参考边实现跨分支信息流，并采用熵启发的渐进式调度，将搜索从广泛探索逐步转向聚焦开发。为了使智能体能够随累积经验进行演化，我们引入了回顾性记忆，它结合了冷启动领域知识库和动态全局记忆，用于任务特定经验的检索与重用。为了实现稳定的长期迭代，我们进一步将战略规划与自适应编码模式的代码生成解耦。在MLE-Bench上的评估表明，MLEvolve在多个维度上达到了最先进性能，包括在12小时预算（标准运行时的一半）下的平均奖牌率和有效提交率。此外，MLEvolve在数学算法优化任务上还超越了包括AlphaEvolve在内的专用算法发现方法，展示了强大的跨领域泛化能力。我们的代码可在https://github.com/InternScience/MLEvolve获取。

## Abstract
Large language model (LLM) agents are increasingly applied to long-horizon tasks such as scientific discovery and machine learning engineering (MLE), where sustained self-evolution becomes a key capability. However, existing MLE agents suffer from inter-branch information isolation, memoryless search, and lack of hierarchical control, which together hinder long-horizon optimization. We present MLEvolve, an LLM-based self-evolving multi-agent framework for end-to-end machine learning algorithm discovery. By extending tree search to Progressive MCGS, MLEvolve enables cross-branch information flow through graph-based reference edges and gradually shifts the search from broad exploration to focused exploitation with an entropy-inspired progressive schedule. To allow the agent to evolve with accumulated experience, we introduce Retrospective Memory, which combines a cold-start domain knowledge base with a dynamic global memory for task-specific experience retrieval and reuse. For stable long-horizon iteration, we further decouple strategic planning from code generation with adaptive coding modes. Evaluation on MLE-Bench shows that MLEvolve achieves state-of-the-art performance across multiple dimensions including average medal rate and valid submission rate under a 12-hour budget (half the standard runtime). Moreover, MLEvolve also outperforms specialized algorithm discovery methods including AlphaEvolve on mathematical algorithm optimization tasks, demonstrating strong cross-domain generalization. Our code is available at https://github.com/InternScience/MLEvolve.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：在长期（long-horizon）的自动机器学习工程任务中，如端到端算法设计、科学发现等，智能体需要持续自我进化——不断生成方案、执行代码、评估结果并调整策略。然而现有MLE智能体存在三大关键缺陷：
  - **分支间信息隔离**：传统树搜索（如MCTS）中，不同搜索分支无法相互借鉴成功经验；
  - **无记忆搜索**：搜索过程仅传递标量奖励，缺乏对历史尝试的积累与复用；
  - **缺乏层级控制**：规划与代码生成耦合在一体，每次迭代重写整个解决方案，效率低且不可控。
- **整体含义**：本文旨在构建一个能自我进化的多智能体框架，以解决上述问题，实现端到端的机器学习算法自动发现，并验证其跨领域泛化能力。

## 2. 论文提出的方法论

- **核心思想**：提出 **MLEvolve**，一个基于LLM的自演化多智能体框架，由三个核心组件构成：
  1. **渐进式蒙特卡洛图搜索（Progressive MCGS）**  
     - 将标准MCTS扩展为有向图结构：节点代表候选解；边分为主边（生成关系）和参考边（跨分支信息流）。
     - 引入**熵启发渐进式调度**：随搜索时间t，探索常数c(t)逐步减小，并通过软开关在UCT探索与精英引导利用之间切换，使有效分支数从早期~4.8降至后期~2.8，实现从广泛探索到聚焦开发的过渡。
     - 扩展操作类型：包括主扩展、分支内演化（回顾同支历史）、跨分支参考（引入其他分支优秀节点）、多分支聚合（融合多分支轨迹创建新起点）。
     - 停滞检测：分支级停滞触发分支内演化/跨分支参考；全局停滞触发多分支聚合。
  2. **回顾性记忆（Retrospective Memory）**  
     - 包含静态领域知识库（冷启动时提供任务相关模型与指南）和动态全局记忆（自动累积每次有效执行的计划、代码、指标、分析等）。
     - 混合检索：BM25（词汇匹配）+ FAISS（语义搜索）→ RRF融合→ Top-K记录。
     - 阶段感知检索：规划阶段以计划为查询；调试阶段以错误消息为查询。
  3. **层级规划与自适应代码生成**  
     - 解耦规划器（决定“改什么、为什么改”）与编码器（决定“怎么改”）。
     - 三种编码模式：Base模式（全量重写）、Stepwise模式（按模块依次生成）、Diff模式（补丁编辑）。根据搜索状态自适应选择。

- **关键技术细节**：
  - 奖励设计：执行失败得-1，成功但不提升分支最优得1，成功且刷新分支最优得2。
  - 反向传播仅沿主边进行，参考边不参与。
  - 精英引导利用：从全局Top-K节点中按排名倒数加权选择。
  - 整个框架通过专门的智能体团队实现（草稿智能体、改进智能体、调试智能体、演化智能体、融合智能体、聚合智能体、代码审查智能体、数据泄露检测智能体、结果解析智能体）。

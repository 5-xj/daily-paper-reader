## 摘要
基于大型多模态模型（LMM）的生成式列表排序旨在单次前向传递中捕获全局列表上下文，但在长上下文多模态场景中其效果会下降。我们识别出一种重复出现的故障模式——解析崩溃，即自回归解码器通过静默省略候选并提前终止生成流畅但不完整的排序。该故障源于有限的上下文利用而非简单的格式错误，使得提示工程和约束解码不足以解决。我们提出PRISMR（参数化表示内化用于语义多模态排序），该框架用参数化结构条件替代瞬时的上下文内列表处理。PRISMR使用轻量级超网络并行编码多模态候选并生成特定于项的LoRA权重，这些权重被合成为LMM的实例特定适配器。这种范式在保留基础模型的同时，实现了更稳健的列表结构内化。我们进一步引入了一个大规模多模态评论排序基准用于评估。实验表明，PRISMR显著减少了解析崩溃，提升了列表排序性能，并在不同领域和指令微调基座上有效迁移。

## Abstract
Generative listwise ranking with Large Multimodal Models (LMMs) aims to capture global list context in a single forward pass, but   its effectiveness degrades in long-context multimodal scenarios. We identify a recurring failure mode, parse collapse, where the   autoregressive decoder produces fluent yet incomplete rankings by silently omitting candidates and terminating early. This   failure stems from limited context utilization rather than simple formatting mistakes, making prompt engineering and constrained   decoding insufficient. We propose PRISMR (Parameterized Representation Internalization for Semantic Multimodal Ranking), a   framework that replaces transient in-context list processing with parametric structural conditioning. PRISMR uses a lightweight   hypernetwork to encode multimodal candidates in parallel and generate item-specific LoRA weights, which are synthesized into an   instance-specific adapter for a LMM. This paradigm enables more robust internalization of list structure while preserving the   base model. We further introduce a large-scale multimodal review-ranking benchmark for evaluation. Experiments demonstrate that   PRISMR substantially reduces parse collapse, improves listwise ranking performance, and transfers effectively across domains and   instruction-tuned backbones.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题**：大型多模态模型（LMM）在进行生成式列表排序（listwise ranking）时，当候选列表较长且包含多模态内容（文本+图像）时，会出现**解析崩溃（parse collapse）**——模型自回归解码生成流畅但**不完整的排序**，表现为静默省略候选、提前终止输出，从而导致严重的幻觉、缺失候选和无法解析的输出。
- **背景**：传统点式排序（pointwise）忽视列表内交互，成对排序（pairwise）复杂度为 O(N²)，而生成式列表排序理论上可通过单次前向传递利用全局上下文，但长多模态上下文加剧了注意力稀释和“中间迷失”现象，导致解析崩溃。
- **现有方法不足**：提示工程（如RankGPT、提示压缩LLMLingua）和约束解码（强制输出格式）对解析崩溃改善有限，因为它们仅处理格式问题而非根本的上下文利用瓶颈；标准微调（SFT）和直接偏好优化（DPO）永久改变全局权重，难以动态适应实例级别的多模态上下文；上下文蒸馏（CD）需逐提示梯度更新，计算成本过高。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
将候选列表信息从**脆弱的上下文提示（in-context）** 转移到**参数化结构空间**，通过一个轻量超网络为每个候选生成项特定的LoRA适配器，再合成一个实例级适配器注入冻结的LMM，从而在解码前编码列表结构先验，减少上下文负担。

### 关键技术细节
1. **超网络（Hypernetwork）Hφ**：
   - 共享的6层Perceiver风格交叉注意力编码器，128个潜在token，宽度1024。
   - 输入：每个多模态候选 d_i = (文本 t_i, 视觉 v_i) 经基础模型的tokenizer和视觉编码器提取特征。
   - 输出：针对LMM中所有36个Transformer层的 {q, k, v, o} 投影，生成LoRA因子 (A_i^(ℓ), B_i^(ℓ))，秩 r=2。
   - 所有候选并行生成，互不依赖。

2. **适配器合成（Adapter Synthesis）**：两种零成本测试时合成模式，来自同一个训练好的超网络。
   - **α模式（α-mode）**：沿秩维度拼接 LoRA 因子，Aα=[A1;...;AN]，Bα=[B1,...,BN]，最终增量 ΔWα = Σ_i B_i A_i。有效秩 ≤ N·r，提供更高容量，适合**训练时见过的列表长度（N ≤ 50）**。
   - **β模式（β-mode）**：均值池化，ΔWβ = (1/N) Σ_i B_i A_i。有效秩 ≤ r，更新尺度有界，对**长度外推（N > 50）** 更鲁棒。
   - 默认规则：N≤50用α模式，N>50用β模式。

3. **训练损失**：
   - 教师模型（GPT-5.4）产生结构化得分序列 y*。
   - 令牌级蒸馏损失 L_NLL：最小化冻结LMM加上合成适配器后生成教师序列的负对数似然。
   - 排序感知损失 L_rank：基于LambdaRank-NDCG，对得分线令牌的平均对数概率作为差异化代理得分，强调对NDCG影响大的错误排序对。
   - 总损失 L_total = L_NLL + λ L_rank，λ=0.5，仅训练超网络参数。

### 公式/算法流程（文字描述）
1. 输入候选列表 D = {d_1,...,d_N}，每个包含文本和图像。
2. 共享超网络 Hφ 并行编码每个候选，生成项特定LoRA因子 (A_i, B_i)。
3. 根据列表长度选择合成模式：若N≤50，采用α模式拼接所有项；若N>50，采用β模式均值池化。得到单一权重增量 ΔW。
4. 将 ΔW 注入冻结的LMM，解码时模型仅需处理简短触发提示（trigger prompt）而非完整多模态列表。
5. 训练时，通过L_NLL和L_rank优化超网络；推理时直接使用合成适配器进行列表生成。

## 6. 论文的主要结论与发现

- **解析崩溃可克服**：PRISMR将候选信息从提示转移到参数空间后，在线演算列表中始终达到≥99.9%的解析率，而Base模型解析率随N增加急剧下降（如N=50时仅2.0%）。
- **排序质量大幅提升**：PRISMR默认模式在所有N下NDCG@10均显著优于所有基线，且优势随N增大而扩大（例如N=50时，PRISMR比最强基线Constrained-decoding高0.158 NDCG@10）。
- **长度泛化与效率双优**：α模式在训练范围内最优，β模式在外推时更鲁棒；PRISMR的推理延迟显著低于约束解码（N=50时加速1.9倍）。
- **点式排序也受益**：PRISMR在点式设置中同样提升NDCG，且在SFT基座上仍有增益。
- **跨领域迁移有效**：从Baby Products到Amazon Fashion零样本转移，解析率保持100%，NDCG@10接近域内水平（0.982 vs 0.974），说明解析崩溃是解码协议的问题而非域特定。
- **合成模式权衡**：α模式容量更高但外推不稳定，β模式更鲁棒但容量受限；联合使用实现最佳权衡。

## 7. 优点（方法或实验设计亮点）

- **问题识别精准**：首次明确指出并系统分析多模态列表排序中的“解析崩溃”故障模式，并证明其非格式错误。

- **框架通用性**：不仅适用于列表排序，在点式设置中也有效；跨领域零样本迁移成功，且可应用于新领域微调后进一步提升。

## 8. 不足与局限

- **格式合规性为经验性而非理论保证**：虽然在线演算范围内解析率≥99.9%，但α模式在长外推时性能急剧下降（N=100时解析率85.1%，NDCG@10仅0.335），因此必须依赖β模式切换，缺乏理论边界。
- **评估基础模型单一**：所有主要实验仅基于Qwen3-VL-8B，未验证其他LMM（如Gemini、GPT-4V）上的泛化性。
- **跨领域测试范围有限**：仅从Baby Products迁移到Amazon Fashion，且后者属于同类电商评论场景，对差异更大的领域（如医疗、法律文档）未验证。
- **排序主观性**：评论排序本身主观，基准虽然通过人工验证，但无法完全代表所有用户偏好，尤其当候选质量接近时。
- **算力需求较高**：超网络含5.8亿参数，需要B200 GPU训练数小时，对资源有限团队可能门槛较高。
- **未与基于logits的重排序器（如FIRST、RankZephyr）对比**：这些方法规避了解析问题但可能面临其他瓶颈。

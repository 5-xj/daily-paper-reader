## 摘要
移动目标旅行商问题（MT-TSP）寻求一条最小成本轨迹，该轨迹使代理从静态仓库出发，在分配的时间窗内访问一组移动目标，然后返回仓库。本文研究了带有移动障碍物的移动目标旅行商问题（MT-TSP-MO），这是MT-TSP的推广，其中代理轨迹必须避开移动障碍物。我们提出了一种混合整数锥规划（MICP）公式，可使用现成求解器求解，以及一种快速且可扩展的两阶段双层搜索（TPBS）算法，该算法为问题计算高质量可行解。我们在包含多达40个目标和40个障碍物的广泛问题实例上，将我们的方法与现有基线算法进行了评估。结果表明，所提出的两种方法在成功率、解决方案成本和计算时间方面均显著优于基线方法。

## Abstract
The Moving-Target Traveling Salesman Problem (MT-TSP) seeks a minimum cost trajectory for an agent that departs from a static depot, visits a set of moving targets, each within one of their assigned time windows, and returns to the depot. In this article, we study the Moving-Target Traveling Salesman Problem with Moving Obstacles (MT-TSP-MO), a generalization of the MT-TSP where the agent trajectory must avoid moving obstacles. We present a Mixed-Integer Conic Programming (MICP) formulation that can be solved using off-the-shelf solvers, as well as a fast and scalable Two-Phase Bilevel Search (TPBS) algorithm that computes high-quality feasible solutions for the problem. We evaluate our approaches against an existing baseline algorithm on a broad range of problem instances with up to 40 targets and 40 obstacles. The results demonstrate that both the proposed methods significantly outperform the baseline with respect to success rates, solution costs, and computation time.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- **混合整数锥规划（MICP）**：将问题建模为混合整数二阶锥规划，利用时间离散化和逻辑约束（如障碍物避让的big-M约束）精确描述问题，可使用Gurobi等商业求解器求解，具有与时间离散化相关的渐进完备性和最优性保证。
- **两阶段双层搜索（TPBS）**：分为**构造阶段**（Phase 1）和**改进阶段**（Phase 2）。在每个阶段内部，采用**双层搜索框架**：高层搜索在广义旅行商问题（GTSP）图上求解候选路线，低层搜索逐一评估路线中每条边的可行性（碰撞检测和时间可行性），若直线连接不可行则在低层时间扩展图中进行DFS寻径。

### 关键技术细节
- **高层GTSP图（\(G_H\)）**：节点为每个目标在各自时间窗内采样的轨迹点，以及仓库节点；边成本为两点间时间差（若速度可行），否则为无穷大；需满足每个簇（对应同一目标）恰好访问一次。
- **低层图（\(G_L\)）**：时间扩展图，包含障碍物自由空间中的采样点和目标轨迹点，边连接相邻时间步的节点，需满足速度约束和障碍物避让（多边形半平面检查）。
- **Phase 1**：使用深度优先搜索（HighLevelDFS）生成初始GTSP候选路线，随后通过EvalEdge逐一评估边；若路线可行则加入初始集合，并对该路线使用的时间窗转换增加惩罚（δ=5）以促进多样性；最多生成N_tours=4条初始路线。
- **Phase 2**：按初始路线成本排序，依次调用GTSP改进启发式（如PGLNS）获得候选改进路线，若新边尚未评估则执行低层搜索；当所有边均已评估且无法进一步降低成本时停止，将改进路线加入最终集合。
- **最终输出**：从初始和改进集合中选择成本最低的可行路线，通过GenAgentTraj生成连续轨迹。

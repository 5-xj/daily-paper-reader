## 摘要
多模态大语言模型最近通过下一个标记预测直接建模查询-文档相关性，展现出作为逐点重排序器的强大潜力。然而，逐点重排序面临跨查询-文档对的显著重复计算问题，而Transformer的因果结构仅允许通过预缓存重用前缀片段。为了解决现有查询优先和文档优先格式与VQA风格提示及计算感知重用的不对齐，我们提出了一种"视觉优先"表述，同时提高了缓存重用效率和重排序性能。然而，剩余成本仍然相当可观，主要来源于三个方面：(1) 模型深度，我们通过早停法减少活跃参数；(2) 跨段注意力，我们将其限制在少数层的窄交互带内；(3) 视觉标记，我们通过嵌入器引导剪枝减少标记数量。这些设计共同构成了miniReranker，在单个查询的高重用设置下，将重排序运行时间减少到密集实现的<1%，同时保持>96%的密集模型性能。

## Abstract
Multimodal large language models (MLLMs) have recently shown strong potential as point-wise rerankers by directly modeling query--document relevance through next-token prediction. However, point-wise reranking suffers from substantial repeated computation across query--document pairs, while the causal structure of transformers allows only prefix segments to be reused via pre-caching. To address the misalignment of existing query-first and document-first formats with both VQA-style prompting and computation-aware reuse, we propose a \textit{vision-first} formulation that improves both cache reuse efficiency and reranking performance. However, the remaining cost is still considerable and stems from three main sources: (1) \textit{model depth}, for which we reduce active parameters via early exit; (2) \textit{cross-segment attention}, which we restrict to a narrow interaction band across a few layers; and (3) \textit{visual tokens}, where we reduce the number of tokens via embedder-guided pruning. Together, these designs form miniReranker, which reduces reranking runtime to <1% of the dense implementation under high-reuse settings for a single query, while preserving >96% of the dense model performance.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：多模态大语言模型（MLLM）作为逐点重排序器（point-wise reranker）时，需要为每个查询与候选文档独立进行精细的 token 级别交互，导致大量重复计算。特别是在检索系统“召回-重排”两阶段流水线中，一个查询往往需要与数十乃至上百个候选文档配对，显存和计算开销巨大。
- **现有方法缺陷**：现有的 MLLM 重排序器普遍继承文本检索领域的“查询优先”序列格式（`[ins, q, d]`），但这种顺序与 MLLM 的 VQA 训练格式（视觉在前、文本在后）不匹配，且视觉输入（图像/视频）的计算代价远高于文本，导致昂贵的视觉表示被反复编码，严重浪费算力。
- **论文目标**：设计一种高效的多模态重排序框架，通过减少模型深度、跨段交互和视觉 token 三个层面的冗余，在保持重排序质量的同时大幅降低计算成本。

## 2. 方法论：核心思想与关键技术细节

### 2.1 视觉优先提示重组（Vision-First Prompt Reformulation）

- **核心思想**：动态地将**视觉模态置于文本模态之前**，利用 MLLM 因果架构的前缀缓存（prefix KV cache）特性，使视觉表示可被多次复用。
- **具体策略**：
  - **文本→图像（T→V）**：采用文档优先顺序 `[ins, d, q]`，视觉文档可离线缓存。
  - **图像→文本（V→T）**：采用查询优先顺序 `[ins, q, d]`，视觉查询只编码一次并广播。
  - **图像→图像（V→V）**：文档优先，使静态文档库可离线预缓存。
- **理论节省**：通过精确的 FLOPs 公式对比，证明了相比于固定顺序，视觉优先能消除 K 次重复编码重视觉输入的开销（K为候选数）。

### 2.2 模型级压缩（Model-Level Compression）

基于对重排序过程中冗余的系统分析，论文提出三个互补策略：

#### a) 早停（Early Exit）
- **依据**：通过 logit lens 逐层探测发现，重排序信号在中层即趋于饱和，约前 60% 层即可达到最终层 95% 的性能（图2）。
- **做法**：在预定义层（2B 模型在第16层，4B/8B 在第21层）截断前向传播，减少活跃参数至原模型的 57%~58%。

#### b) 交互带（Interaction Band）
- **依据**：通过掩码实验发现，跨段注意力只有在少数中间层才发生有效的信息融合（图2的相似度骤降区域）。定义该狭窄的层范围（如 2B 模型第 8-16 层）为“交互带”。
- **做法**：仅在交互带内启用查询-文档注意力，其余层保持段内自注意力，从而大幅减少二次注意力计算量。

#### c) 嵌入器引导的视觉 token 剪枝（Embedder-Guided Token Pruning）
- **依据**：大量视觉 token 对最终相关性判决贡献很小。论文利用第一级召回阶段的检索嵌入器（如 Qwen3-VL-Embedding）在召回过程中已经计算好的注意力分数（从最后 token 到各视觉 token 的注意力权重），跨层聚合获得重要性排名。
- **做法**：保留前 α% 的高重要性 token（实验中设为 50%），丢弃其余 token，无需额外前向传播。表2显示该剪枝策略优于单层或简单相似度选择。

### 2.3 训练设置

- 在 Qwen3-VL-Instruct 基础上，使用 LoRA（r=16, α=32）微调 LLM 部分，训一个 epoch。
- 构建约 736K 对点式重排序数据集，含正例和负例（硬负+随机负）。

## 6. 主要结论与发现

1. **视觉优先提示** 同时提升重排序准确度和缓存复用效率，比固定 QF/DF 格式更好。
2. 重排序信号在中层即饱和，早停在 60% 深度处可保留 >96% 性能。
3. 跨段注意力仅在狭窄的中间层有效，交互带可安全消除其余层的高成本注意力。
4. 利用检索阶段嵌入器的注意力分数进行零开销视觉 token 剪枝，可保留半数 token 而不损失质量。
5. **miniReranker 整体效果**：在 2B/4B/8B 模型上分别保留 96.3%、98.3%、98.6% 的密集基线性能；活跃参数降至 58%；训练加速近 3×；视频重排序延迟降至 <1%，图像降至 <15%（单查询 Top-100 高重用设置）。

## 7. 优点

- **系统性分析**：通过 logit lens、注意力掩码实验、余弦相似度分析等方法，科学地揭示了模型深度、交互密度、token 数量三个层面的冗余来源，而非盲目压缩。
- **零开销剪枝**：嵌入器引导的 token 剪枝完全复用已有计算，无需额外前向或外部模型，工程实现巧妙。
- **正交性设计**：早停、交互带、剪枝各自针对不同维度的冗余，可叠加使用，效果互补。
- **开源友好**：基于 Qwen3-VL 并公开训练数据集构建细节，可复现性强。
- **实验全面**：覆盖 78 个多模态任务，效率分析包含多场景缩放，结论可靠。

## 8. 不足与局限

- **适用范围**：仅针对逐点式（point-wise）重排序，未推广到列表式（list-wise）或基于 agent 的检索管线，这些场景下跨文档交互更复杂。
- **架构依赖**：依赖 MLLM 因果结构与 prefix caching，其他非因果或双向编码器架构可能不适用。
- **对视觉输入长度的敏感性**：剪枝策略固定为保留 50%，对于极长视频或超高清图像，可能仍需自适应比例。

- **潜在偏差**：训练数据中负样本由同一嵌入器召回，可能引入分布偏差；且论文未讨论重排序结果的公平性（如针对不同语言或领域）。

## 摘要
多任务学习（MTL）在推荐系统中至关重要，能够实现不同用户反馈之间的互补学习。虽然现代工业实践已从DNN转向以Transformer为中心的架构以增强序列建模和扩展能力，但它们仍然将特征编码与多任务预测解耦，将Transformer视为任务无关的编码器。这种设计从根本上限制了性能和可扩展性，因为（1）在异构任务目标下造成信息瓶颈，（2）引发梯度干扰导致跷跷板现象，（3）强制数据流转换，使得基于注意力、上下文自适应的表示学习被转化为具有不兼容信息读写动态的静态前馈任务预测。我们提出OneRank，一种原生Transformer多任务排序框架，消除了编码器-预测器的分离，并为前向表示学习和反向优化引入了任务私有通道，实现了任务专用学习同时减少任务间干扰。在前向传播中，OneRank通过任务条件信息选择、候选感知上下文化和受控的跨任务交互，自底向上学习任务特定表示。在反向传播中，跨任务梯度分离将任务私有参数更新与共享知识提取模块隔离，防止负迁移。我们进一步用基于动态匹配的评分替代静态任务特定MLP评分器，实现上下文感知的个性化排序。通过将多任务推理内化到Transformer堆栈中，OneRank建立了一个统一且可扩展的架构范式。在大型工业数据集上的离线和在线实验表明，OneRank在保持计算效率的同时显著优于最先进的基线方法。

## Abstract
Multi-task learning (MTL) is essential in recommender systems to enable complementary learning among diverse user feedback. While modern industrial practices have shifted from DNNs to Transformer-centric architectures to strengthen sequence modeling and scaling capacity, they still decouple feature encoding from multi-task prediction, treating the Transformer as a task-agnostic encoder. This design fundamentally limits the performance and scalability by (1) creating an information bottleneck under heterogeneous task objectives, (2) inducing gradient interference that leads to the seesaw phenomenon, and (3) forcing a dataflow transition in which attention-based, context-adaptive representation learning is converted to static feed-forward task prediction with incompatible information read-write dynamics.   We propose OneRank, a Transformer-native multi-task ranking framework that eliminates encoder-predictor separation and introduces task-private channels for forward representation learning and backward optimization, enabling task-specialized learning while reducing inter-task interference. In the forward pass, OneRank learns task-specific representations bottom-up through task-conditioned information selection, candidate-aware contextualization, and controlled cross-task interaction. In the backward pass, cross-task gradient detachment isolates task-private parameter updates from shared knowledge extraction modules, preventing negative transfer. We further replace static task-specific MLP scorers with dynamic matching-based scoring for context-aware personalized ranking. By internalizing multi-task reasoning within the Transformer stack, OneRank establishes a unified and scalable architectural paradigm. Offline and online experiments on large-scale industrial datasets show that OneRank significantly outperforms state-of-the-art baselines while maintaining computational efficiency.

# OneRank: 面向多任务推荐的原生 Transformer 排序架构 — 详细总结

## 1. 核心问题与整体含义

- **背景**：现代推荐系统广泛采用多任务学习（MTL）来联合建模密集但噪声多（如点击）与稀疏但信息量高（如购买）的用户反馈。近年来，工业界从 DNN 转向 Transformer 以增强序列建模和扩展能力。
- **核心问题**：现有 Transformer 在 MTL 推荐中仍保留“编码器–预测器”分离范式（`G(Z = F(X))`），即将 Transformer 仅作为任务无关的编码器，再接入独立的任务塔（MLP）。该设计存在三个根本缺陷：
    1. **任务无关信息瓶颈**：共享表示 `Z` 将任务特定信号与共享知识纠缠，迫使下游预测器在容量有限的 MLP 中做复杂解耦。
    2. **梯度干扰（跷跷板现象）**：共享参数上的冲突梯度导致一个任务提升而另一个任务下降。
    3. **数据流范式不匹配**：Transformer 注意力的上下文自适应学习与 MLP 的静态前馈预测之间信息读写动态不一致，破坏了端到端推理和计算缩放。
- **本文意义**：提出了 OneRank，首个将多任务推理完全内化到 Transformer 堆栈中的统一排序框架，从根本上消除了编码器–预测器分离，实现了任务专用学习、稳定优化和动态上下文感知排序。

## 2. 方法论

### 核心思想
通过内化多任务推理到 Transformer 架构中，同时构建任务私有通道和任务共享通道，实现自底向上的任务专用表示学习与受控跨任务知识迁移。

### 关键技术细节（前向 + 反向 + 预测）

1. **结构化分词（Structured Tokenization）**
   - 将异构输入统一为 token 序列：
     - 用户交互历史（IH）+ 偏好锚点（PA，类似 RAG 检索的序列）。
     - 对每个候选物品，创建一个候选-任务组：`[候选嵌入, 任务1 token, 任务2 token, ...]`。
   - 最终序列：`[SIH, SPA, G1, G2, ..., GN]`。

2. **任务特定编码（Task-Specific Encoding）**
   - 引入**任务特定 token 互不可见**：同一候选组内不同任务 token 只能关注用户上下文和本候选嵌入，不能关注其他任务 token。
   - 结构化注意力掩码实现候选组隔离、因果用户上下文、任务 token 互不可见，从而在底层强制早期任务专业化。

3. **候选感知上下文化（Candidate-Aware Contextualization）**
   - 定义**情景描述符（s）**：包含用户/查询/会话元数据。
   - 通过任务特定的投影函数 `f_k(s)` 得到查询 `q_k`，再对每个任务使用独立的多头交叉注意力（MHCA_k）聚合所有候选的表示，得到任务级全局表示 `h_k`。
   - 显式建模跨候选竞争依赖，弥合训练-服务差距。

4. **多任务预测（Multi-Task Prediction）**
   - **跨任务关系注意力**：对 `{h_k}` 施加可配置掩码 `A`（并行、空、级联、混合），允许或阻止任务间信息流。
   - **战略梯度分离（Gradient Detachment）**：反向传播时仅允许对角线梯度流，阻断非对角线梯度，使跨任务注意力变成“只读存储器”，保留前向知识迁移但消除负干扰。
   - **动态匹配评分**：用内积 `s_{ik} = z_k^T r_{ik}` 替代静态 MLP，其中 `z_k` 是经跨任务交互和 FFN 精化的全局表示，`r_{ik}` 是候选的任务特定表示；实现上下文自适应排序。

5. **联合学习目标**
   - InfoNCE 列表损失 + BCE 点损失联合优化。

## 7. 优点

- **架构创新**：彻底消除了编码器–预测器分离，实现了原生 Transformer 的多任务推理，是范式级别的改进。
- **任务专用与共享兼顾**：通过任务 token 互不可见、独立跨候选聚合、梯度分离等机制，在不失知识迁移的前提下有效抑制了跷跷板现象。
- **动态上下文感知**：用内积匹配替代静态 MLP，使评分随会话上下文自适应调整，更符合实际排序需求。
- **灵活性**：可配置的跨任务掩码（并行/空/级联/混合）允许轻松编码领域知识，无需重新设计模型。
- **工业实用性**：在真实大规模平台上验证，离线在线均取得显著提升，且部署时通过分组并行和 KV-cache 保持了低延迟。

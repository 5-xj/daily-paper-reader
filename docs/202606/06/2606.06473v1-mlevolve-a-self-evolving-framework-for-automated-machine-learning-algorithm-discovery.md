## 摘要
大型语言模型（LLM）智能体正越来越多地应用于长时域任务，如科学发现和机器学习工程（MLE），其中持续自我进化成为关键能力。然而，现有的MLE智能体存在分支间信息隔离、无记忆搜索以及缺乏层次化控制等问题，这些问题共同阻碍了长时域优化。我们提出了MLEvolve，一个基于LLM的自进化多智能体框架，用于端到端的机器学习算法发现。通过将树搜索扩展为渐进式MCGS，MLEvolve利用基于图的参考边实现跨分支信息流动，并借助熵启发的渐进式调度，逐步将搜索从广泛探索转向聚焦利用。为了让智能体能够随着积累的经验进化，我们引入了回顾记忆（Retrospective Memory），它将冷启动领域知识库与动态全局记忆相结合，用于任务特定经验的检索和重用。为了实现稳定的长时域迭代，我们进一步将战略规划与代码生成解耦，并采用自适应编码模式。在MLE-Bench上的评估显示，MLEvolve在包括平均奖牌率和在12小时预算（标准运行时的一半）内的有效提交率等多个维度上达到了最先进的性能。此外，MLEvolve在数学算法优化任务上也优于专门的算法发现方法（包括AlphaEvolve），展现出强大的跨领域泛化能力。我们的代码可在 https://github.com/InternScience/MLEvolve 获取。

## Abstract
Large language model (LLM) agents are increasingly applied to long-horizon tasks such as scientific discovery and machine learning engineering (MLE), where sustained self-evolution becomes a key capability. However, existing MLE agents suffer from inter-branch information isolation, memoryless search, and lack of hierarchical control, which together hinder long-horizon optimization. We present MLEvolve, an LLM-based self-evolving multi-agent framework for end-to-end machine learning algorithm discovery. By extending tree search to Progressive MCGS, MLEvolve enables cross-branch information flow through graph-based reference edges and gradually shifts the search from broad exploration to focused exploitation with an entropy-inspired progressive schedule. To allow the agent to evolve with accumulated experience, we introduce Retrospective Memory, which combines a cold-start domain knowledge base with a dynamic global memory for task-specific experience retrieval and reuse. For stable long-horizon iteration, we further decouple strategic planning from code generation with adaptive coding modes. Evaluation on MLE-Bench shows that MLEvolve achieves state-of-the-art performance across multiple dimensions including average medal rate and valid submission rate under a 12-hour budget (half the standard runtime). Moreover, MLEvolve also outperforms specialized algorithm discovery methods including AlphaEvolve on mathematical algorithm optimization tasks, demonstrating strong cross-domain generalization. Our code is available at https://github.com/InternScience/MLEvolve.

## 2. 方法论

### 核心思想
MLEvolve将算法发现形式化为在结构化搜索空间中寻找最优解的过程，并设计三个核心组件协同工作：**Progressive MCGS**（渐进式蒙特卡洛图搜索）解决分支隔离和固定探索策略问题；**Retrospective Memory**（回顾记忆）实现经验积累与重用；**Hierarchical Planning with Adaptive Code Generation**（层次化规划与自适应代码生成）解耦规划与实现，提高迭代稳定性。

### 关键技术细节

#### 2.1 Progressive MCGS
- **搜索空间构建为有向图**：$G = (V, E), E = E_T \cup E_{\text{ref}}$，其中$E_T$为主边（父-子生成关系），$E_{\text{ref}}$为参考边（跨分支或非邻接节点的信息流）。
- **渐进式探索调度**：采用软切换机制，根据时间$t$的概率选择UCT策略（高熵探索）或Elite-Guided策略（低熵利用）：$P(S_t = \text{UCT}) = w(t), P(S_t = \text{Elite}) = 1 - w(t)$，其中$w(t)$从1.0渐减到$w_{\min}$。Elite-Guided模式从全局Top-K节点中按逆秩加权采样。
- **四种扩展类型**：
  1. **Primary expansion**（$R=\emptyset$）：标准父-子生成。
  2. **Intra-branch evolution**（$R = \mathcal{R}_{\text{hist}}(v_t,k)$）：基于同一分支最近$k$个节点的轨迹进行自我反思。
  3. **Cross-branch reference**（$R = \mathcal{R}_{\text{cross}}(N)$）：分支停滞时，引入其他分支的Top-N节点作为参考。
  4. **Multi-branch aggregation**（$R = \mathcal{R}_{\text{agg}}$）：全局停滞时，聚合多个分支的最佳轨迹，从根节点创建新分支。
- **奖励与反向传播**：执行结果分为-1（失败）、1（成功但未改善）、2（刷新分支最佳），仅沿主边$E_T$反向传播。

#### 2.2 Retrospective Memory
- **领域知识库**：冷启动时提供按任务类型组织的候选模型和用法指南，减少初始错误。
- **动态全局记忆**：每次有效节点执行后，自动记录计划、代码、指标、分析等结构化经验。检索采用混合方式：BM25（词法）+ FAISS（语义）+ RRF（倒数排名融合）得到Top-K记录。按规划阶段和调试阶段设置不同查询。

#### 2.3 Hierarchical Planning with Adaptive Code Generation
- **Planner-Coder解耦**：Planner负责模块级别的战略决策（修改什么、为什么），Coder负责代码级别的实现（具体如何修改）。
- **三种编码模式**：
  - **Base模式**：从头生成完整代码（用于初始草稿）。
  - **Stepwise模式**：按模块逐步生成（用于复杂多阶段流水线）。
  - **Diff模式**：基于补丁进行局部编辑（用于已有可行方案的稳定优化）。模式根据搜索状态选择。

## 摘要
符号回归旨在从数据中发现明确的科学定律。近期方法利用大型语言模型从背景文本引导变异，这比随机遗传编程更具方向性。然而，精确的符号恢复需要同时具备语义引导和明确结构，以便通过有效的符号表示进行领域知识引导的搜索。当前基于LLM的系统仍然结构盲：它们在不可解释的候选方案中进行选择，缺乏局部变异的明确机制，并且依赖脆弱的系数拟合，这可能导致正确骨架被低估。我们提出FunctionEvolve，一个利用表达式树组织整个搜索的进化框架：结构摘要促进多样化的父代选择，局部树编辑保留有用的子表达式，结构感知的拟合分解、约束和简化系数，以实现更可靠的评分。它仅使用基本函数族，无需额外的领域特定规则限制泛化能力。在LLM-SRBench的129任务合成子集上，使用Claude Opus 4.6的FunctionEvolve恢复了107个精确形式，SA@50达到82.9%，是相同骨干基线的4.5倍；SA@1达到55.8%，是先前公布的最强top-1结果的3.6倍。消融实验表明，结构可见的搜索对于可靠恢复至关重要，LLM引导的改进和结构感知的系数优化作为必要的提议和评分机制。我们还对基准进行了审计，发现其材料科学子集中的共线性导致了可辨识性问题。

## Abstract
Symbolic regression aims to uncover explicit scientific laws from data. Recent methods use LLMs to guide mutation from background text, which is more directed than random genetic programming. However, exact symbolic recovery requires both semantic guidance and explicit structure, so that domain-informed search are carried out through valid symbolic representation. Current LLM-driven systems remain structure-blind: they select among opaque candidates, lack explicit mechanisms for local mutation, and rely on brittle coefficient fitting that can undervalue correct skeletons. We propose FunctionEvolve, an evolutionary framework using expression trees to organize the whole search: structural summaries promote diverse parent selection, local tree edits preserve useful subexpressions, and structure-aware fitting decomposes, constrains, and simplifies coefficients for more reliable scoring. It uses only elementary function families, without additional domain-specific rules limiting generalization. On the 129-task synthetic subset of LLM-SRBench, FunctionEvolve with \emph{Claude Opus 4.6} recovers 107 exact forms, reaching 82.9% SA@50, 4.5x above same-backbone baselines, and 55.8% SA@1, 3.6x above the strongest previously published top-1 result. Ablations show that structure-visible search is central to reliable recovery, with LLM-guided refinements and structure-aware coefficient optimization serving as essential proposal and scoring mechanisms. We also audit the benchmark and show that collinearity in its materials-science subset creates identifiability issues.

## 1. 核心问题与整体含义（研究动机和背景）
- **符号回归（SR）的目标**：从数据中发现可解释的科学定律（显式数学表达式），相较于黑箱模型，SR 提供可解释性、外推能力和知识迁移能力。
- **现有挑战**：
  - 传统遗传规划（GP）虽然使用表达式树，但变异过程随机，缺乏语义引导，搜索效率低。
  - 近年 LLM 驱动的 SR 方法（如 LLM-SR、OpenEvolve、PiT-PO 等）利用任务上下文提供语义引导，但存在三大瓶颈：
    1. **结构盲**：候选表达式被当作不透明字符串或代码，缺乏显式结构信息，难以进行局部编辑。
    2. **父代选择依赖固定启发式**：无法结构化区分不同骨架，导致搜索预算分配不合理。
    3. **系数拟合脆弱**：正确骨架因系数优化效果差而被低估，过早丢弃。
- **论文核心论点**：显式的抽象语法树（AST）结构可以使搜索遵循 SR 的局部编辑本质，同时帮助父代多样性选择和系数拟合，从而大幅提升精确符号恢复率。

## 2. 论文提出的方法论
- **核心思想**：FunctionEvolve 是一个围绕显式表达式树组织的进化框架，融合了 LLM 的语义引导与 AST 的结构化操作，包含五个关键模块：
  1. **Generator（生成器）**：利用 LLM 从任务上下文中提取领域知识和初始种子表达式，为搜索提供结构化起点。
  2. **Selector（选择器）**：基于进化树摘要（包括 AST 结构、参数数量、树深度、训练 NMSE 等）选择多样化的父代表达式，避免陷入局部最优。
  3. **Mutator（变异器）**：
     - **AST 规则变异**：确定性操作（添加/删除子表达式、包裹函数等），基于预定义基本函数库（线性、三角、幂、指数、对数、有理等）。
     - **LLM 引导变异**：利用父代 AST 结构、拟合参数、历史最佳表达式和领域知识，生成非模板的局部 ADD/SUBST 编辑。
  4. **Optimizer（优化器）**：结构感知的系数拟合流水线，包括：
     - 变量投影（分离线性/非线性参数，线性参数用 OLS 求解）
     - 全局搜索（CMA-ES、DE、信任域最小二乘）
     - 局部精化（L-BFGS-B）
     - 参数快照（将幂指数、有理参数快照到最近的整数或简单有理数）
     - 符号简化（合并常数、消除冗余形式）
  5. **进化树管理**：每个节点存储表达式及其拟合状态，通过成熟度（训练 NMSE 低于阈值）控制搜索终止。
- **算法流程**（简化）：
  1. 初始化：Generator 产生种子表达式并优化评分。
  2. 迭代（最多 30 轮）：
     - Selector 选择 5 个父代。
     - 对每个父代，由 AST 规则和 LLM 变异生成候选表达式（去重）。
     - 对每个候选，进行 Optimizer 拟合和退化检查（过拟合、简化等）。
     - 将有效候选加入进化树，标记成熟节点。
  3. 返回按训练 NMSE 排序的候选列表。

## 6. 论文的主要结论与发现
1. **方法有效性**：FunctionEvolve 在 LLM-SRBench 的 129 个任务上，使用 Claude Opus 4.6 达到 SA@50 = 107（82.9%），SA@1 = 72（55.8%），远超同骨干基线（LLM-SR、OpenEvolve 仅 24 SA@50）和先前最佳 PiT-PO（20 SA@1）。
2. **结构可见搜索是核心**：去掉 AST 结构后 SA@50 从 107 降至 60，证明 AST 结构不仅提供规则变异，还帮助 LLM 进行有意义的局部推理。
3. **LLM 引导变异与结构感知优化均至关重要**：分别去掉后 SA@50 降至 46 和 53，组合去掉时降至 34。
4. **纯规则变异 + 结构感知优化已接近先前 SOTA**（No LLM 变体 SA@50=29，SA@1=16，而 PiT-PO top-1 为 20），说明可靠拟合本身就有力。
5. **数值精度接近真实表达式**：Acc_0.1 达 113/129，中位数 NMSE 1.38e-13。
6. **AI-Feynman 上 120/120 完全恢复**，但大部分在种子阶段即匹配，表明存在 LLM 记忆污染风险，因此 LLM-SRBench 才是更干净的测试床。
7. **材料科学子集存在严重共线性问题**：T ≈ 273+500ε，导致温度相关项不可辨识，解释了该领域表现较差的原因。
8. **噪声下性能显著下降**：5% 噪声时 SA@50 从 107 降至 40，表明精确符号恢复对噪声敏感，需要更鲁棒的评分机制。

## 7. 优点（亮点）
- **结构显式化的系统性设计**：将 AST 结构贯穿于选择、变异、拟合，优于现有“结构盲”方法。
- **双通道变异**：规则变异保证局部编辑的确定性，LLM 变异提供语义灵活的非模板编辑，互补性强。
- **结构感知优化器**：通过变量投影、幂指数快照、符号简化等技巧，显著提升系数拟合可靠性，防止正确骨架被低估。这是许多 GP 方法忽视的关键点。
- **效率较高**：相比 LLM-SR，仅用约 1/15 的 LLM 调用数就达到更高性能，且由于并行评估，壁钟时间并未比例增加。
- **严谨的评估协议**：采用双 LLM 裁判 + 人工仲裁符号等价性，避免单一模型误判；审计表明裁判准确率 96-99%。
- **对基准本身的批判性分析**：揭露 LLM-SRBench 数据共线性问题，提升了领域内对基准可靠性的认识。

## 8. 不足与局限
- **对噪声敏感**：当前方法在噪声下恢复率大幅下降，未引入不确定性感知或贝叶斯评分，限制了在真实实验数据上的应用。
- **搜索空间有限**：仅支持基本函数族（多项式、三角、指数、对数等），未涵盖特殊函数（贝塞尔、误差函数等）或微分方程形式，对某些物理定律不适用。
- **LLM 成本**：虽然调用数少，但每次 Selector 调用包含大量上下文，token 消耗高（特别是 Selector 占 85% 的 token），若使用高成本 API 可能不够经济。
- **材料科学子集问题**：虽然作者指出了数据共线性，但 FunctionEvolve 在该子集上的结果仍受影响（SA@50 仅 15/25），方法未能完全克服数据缺陷。
- **未进行跨域泛化测试**：实验限于 LLM-SRBench 和 AI-Feynman，未在更广泛的实际科学数据集（如地震、气候、生物网络）上评估。
- **未考虑动态系统或隐式方程**：只能处理显式 y=f(x) 形式，无法处理微分方程或约束方程。
- **超参数选择**：变异参数（如每父代 20 个 LLM 提议）可能对特定任务不是最优，但未进行系统调优。
- **消融实验“w/o All”接近 LLM-SR，但并行策略不同**：作者指出 wall-clock 成本并不成比例增加，但严格来说实验设置并非完全匹配现有基线，对比公平性略有争议（但作者已做了诚实说明）。

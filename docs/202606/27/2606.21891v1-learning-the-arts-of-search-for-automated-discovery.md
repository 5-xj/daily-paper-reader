## 摘要
科学发现可以表述为在假设和实验空间上的迭代搜索过程。当代方法使用如MCTS等启发式算法来导航这一空间。这些算法将假设的价值与其实验执行质量混为一谈。因此，一个有前景但执行初步的假设被排在一个执行完善但平庸的假设之后。此外，随着搜索进行，历史积累超出上下文窗口，先前方法会裁剪搜索日志。我们提出了一种用于树搜索的推理智能体（ARTS），其中部署了一个推理语言模型来导航该空间。该模型检查先前的执行日志，诊断早期失败源于错误实现还是错误假设，并选择下一步要构建的假设。为缓解上下文长度问题，ARTS使用测试时训练将搜索树的知识灌输到模型权重中。在来自MLGym和MLEBench的22个任务上，我们展示了ARTS优于领先算法，归一化分数相对提升超过15.3%。通过测试时训练，我们展示了Qwen3-4B智能体能够以高达5倍的更低推理成本，匹配如Gemini-3 Pro和GPT o3-reasoning等闭源前沿模型的性能。我们进一步观察到，在部分可观察的强化学习任务中，经过测试时训练的Qwen3-4B科学家通过重新发现启发式方法剪枝掉的人类最佳递归记忆解决方案，超越了使用o3科学家的ARTS。

## Abstract
Scientific discovery can be formulated as an iterative search process over the space of hypotheses and experiments. Contemporary methods navigate this space using heuristics such as MCTS. These algorithms conflate the merit of a hypothesis with the quality of its experimental execution. A promising hypothesis with preliminary execution is therefore ranked below a modest hypothesis whose execution is refined. Moreover, prior methods prune the search logs as the search progresses because the accumulated history outgrows the context window. We propose Agentic Reasoning for Tree Search (ARTS), where we deploy a reasoning language model to navigate this space. The model inspects prior execution logs, diagnoses whether earlier failures arose from faulty implementations or bad hypotheses, and selects the hypothesis to build on next. To mitigate challenges with context length, ARTS uses test-time training to instill the knowledge of search tree in the model weights. Across 22 tasks from MLGym and MLEBench, we show that ARTS outperforms leading algorithms, with over 15.3% relative improvement in the normalized score. With test-time training we show that a Qwen3-4B agent can match performance with closed-source frontier models like Gemini-3 Pro and GPT o3-reasoning with upto 5x lower inference cost. We further observe that on partially observable RL tasks, the test-time trained Qwen3-4B scientist surpasses ARTS with the o3 scientist by rediscovering the human-best recurrent-memory solution that heuristic methods prune away.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：科学发现本质上是一个在假设与实验空间上的迭代搜索过程。现有自动化搜索算法（如MCTS、线性搜索、进化搜索）存在两个关键缺陷：
  1. **混淆假设价值与执行质量**：一个低分节点可能源于实现错误（如代码 bug、欠训练）而非假设本身错误，但基于分数的启发式选择（如 UCB）会错误地抛弃优秀假设。
  2. **超出上下文窗口**：随着搜索树增长，历史日志（代码、执行结果）超过模型上下文限制，现有方法要么直接裁剪导致信息丢失，要么依赖不可靠的长上下文推理。
- **核心目标**：设计一种能够**通过推理区分失败根源**、并**将搜索历史压缩到模型权重中**的搜索算法，从而实现更高效、更鲁棒的自动科学发现。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想

- **Agentic Reasoning for Tree Search (ARTS)**：使用一个**推理语言模型（LRM，作为“科学家”）**来指导搜索，代替传统启发式节点选择。科学家通过**检查节点代码、执行日志、分数**，**推理**该节点失败是因为“假设错误”还是“实现错误”，从而选择最有潜力的节点进行扩展。

### 2.2 关键技术细节

#### 2.2.1 科学家（Scientist）与执行器（Executor）解耦

- **科学家**（默认使用 OpenAI o3）：负责节点选择与假设提议。拥有以下工具：
  - **INSPECT**：读取任意节点的代码、训练曲线、程序输出。
  - **Baseline Audit**（前 R 个科学家调用为审计阶段）：仅读取数据、基线代码、评估脚本，不进行扩展，输出关于任务结构（如度量稀疏性、类不平衡）的发现。
  - **Verbalized Sampling**：科学家枚举 K 个候选假设（h₁...h_K），每个赋予概率 πₖ，由外部采样器按概率抽取一个，避免贪婪采样导致的多样性坍塌。
  - **Memory**：每次扩展后写入一条简短洞察，科学家可随时读取，辅助后续决策。
- **执行器**（默认使用 Gemini 3 Flash）：一个编码智能体，根据科学家的假设和父节点代码状态编写新代码，调用 READ_FILE、WRITE_FILE、VALIDATE 工具，直到得到有效分数或超过预算。

#### 2.2.2 ARTS 算法流程（Algorithm 1）

1. 初始化树 G，包含基线节点。
2. 审计阶段：使用前 R 次科学家调用进行审计（不扩展节点）。
3. 循环直到时间预算耗尽：
   - 科学家 INSPECT 当前树。
   - 科学家输出父节点 pₜ 和 K 个候选假设（每个带概率）。
   - 外部采样器随机抽取一个假设 h′ₜ。
   - 执行器根据父节点代码和假设实现并验证，得到新节点（代码 cₜ、分数 sₜ）。
   - 将新节点加入树。

#### 2.2.3 ARTS*：测试时训练（Test-Time Training）

- **动机**：当树深度增加（如 8 层、分支因子 3 时节点超 6000），科学家模型的上下文窗口无法容纳全部日志。传统方法会裁剪历史，ARTS*用**GRPO（Group Relative Policy Optimization）**在已有搜索树上微调科学家，将搜索经验固化到 LoRA 适配器权重中。
- **流程**（Algorithm 2）：
  - 给定当前树 Gₜ，科学家 πθ 采样 B 个动作（每个动作 = (父节点, 假设)）。
  - 每个动作由执行器实现并产生分数 sᵢ。
  - 计算分位数奖励 rᵢ：
    - null → -0.5
    - < 基线 → -0.2
    - [基线, 70%分位数) → 0
    - ≥ 70%分位数 → +1
  - 使用 GRPO 更新 LoRA 适配器，奖励为单步奖励（非多步轨迹），避免信用分配模糊。
  - 所有 rollouts 加入树，继续下一轮训练。
- **多样性保持**：尽管训练基于分数，因 Verbalized Sampling 仍然保持假设多样性（实验证明训练后多样性不下降）。

1. **性能提升**：ARTS 在 22 个任务中的 16 个上达到最佳自动搜索结果，平均归一化分数比最强基线（MLEvolve）相对提升 **15.3%**，IQM 为 0.93 vs 0.87，最优性差距最小。

## 7. 优点：方法或实验设计上的亮点

- **创新分责**：将“科学家”与“执行器”解耦，分别优化推理和编码能力，是一种智能的分工设计。

- **Verbalized Sampling**：解决多样性坍塌，使搜索覆盖更多方向。

- **实验详尽**：包括模型交换、组件消融、奖励函数、GRPO 结构等，分析深入（如多样性分析通过 TF-IDF、熵、人类标注）。
- **开源代码和轨迹**：提升可复现性。

## 8. 不足与局限

- **依赖模型质量**：ARTS 的性能高度依赖于科学家（如 o3）和执行器（Gemini 3 Flash）的质量。弱推理模型可能无法准确诊断失败。论文虽提出 TTT 可以弥补，但 TTT 训练成本更高。
- **训练成本**：TTT 使用 3 GPU、24 GPU-hours 每运行，虽然推理成本低，但训练开销不低。对于预算有限的团队可能不友好。
- **实验覆盖不完整**：TTT 实验仅限于 MLGym 的 11 个任务，未在 MLEBench 的 11 个任务上验证，可能泛化性不足。
- **预算固定且较短**：8 小时墙钟时间对于某些复杂任务可能不够，且不同任务每节点耗时差异大，ARTS 的探索策略在极短预算下可能不如速度快的基线。
- **安全与伦理风险**：论文本身提到，更强的研究智能体可能被滥用（如生成有害内容），且集中计算资源可能加剧不公平。需要部署时保留人类监督。
- **未讨论失败情况**：论文主要展示成功案例，未深入分析 ARTS 在哪些任务上失败以及为什么（例如 House Price 和 Jigsaw Toxic 性能接近但未超过基线，原因可能是任务饱和）。缺乏对失败模式的系统分析。
- **上下文长度问题仅部分解决**：TTT 蒸馏了搜索树知识，但模型在推理时仍可能受限于描述当前树的 token 数量（需要将树以文本形式输入）。论文未讨论当树本身描述文本也超过上下文时的处理方式。

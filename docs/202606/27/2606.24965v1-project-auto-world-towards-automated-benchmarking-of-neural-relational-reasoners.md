## 摘要
关系结构的推理对于神经模型仍然是一个重大挑战，特别是当它们必须系统地将学到的知识应用于比训练时更困难的问题实例时。由于评估这种泛化能力的困难，进展受到阻碍，因为事先很难明确是什么使一个实例变得困难。我们研究如何通过使用大型语言模型（LLM）来自动生成基准测试来解决这个问题，以端到端的方式学习生成越来越有挑战性的实例。具体来说，给定一个由Datalog规则参数化的世界，以及作为推理评估器的Edge Transformer，我们使用LLM驱动的进化搜索（基于FunSearch）和自主智能体搜索来发现能产生困难问题实例的采样函数。我们还表明，使用这些数据可以改进Edge Transformer，使其能够很好地泛化到进一步的数据扰动。最后，我们展示了同样的机制可以应用于LLM提出的新世界，为神经关系推理的自主研究打开了大门。

## Abstract
Reasoning about relational structures remains a significant challenge for neural models, particularly when they must systematically apply learned knowledge to problem instances that are harder than those seen in training. Progress is hampered by the difficulty of evaluating such generalization, since a priori, it is rarely clear what makes an instance hard. We study how this issue can be addressed by using large language models (LLMs) to automate benchmark generation, learning to produce increasingly challenging instances in an end-to-end manner. Concretely, given a world parametrized by Datalog rules, and an Edge Transformer as the reasoning evaluator, we use LLM-driven evolutionary search (based on FunSearch) and autonomous agentic search to discover sampling functions that yield hard problem instances. We also show that the Edge Transformer can be improved using this data such that it generalizes well to further data perturbations. Finally, we show that the same machinery can be applied to novel worlds proposed by LLMs, opening the door to autonomous research on neural relational reasoning.

## 2. 方法论：核心思想、关键技术细节与算法流程

- **核心思想**：给定一个由 Datalog 规则参数化的“世界”（即领域知识），以及一个已经训练好的神经推理模型（作为评估器，此处为 Edge Transformer），通过 LLM 学习一个**采样函数**（priority function），该函数能指导生成对评估器而言困难的知识图谱（KG）实例。
- **关键技术细节**：
  - **问题形式化**：Difficulty(S; M) = 1 - E_G~S [ min_{h,t} min_r M_r(G, h, t) ]，其中 M 是模型，S 是采样器。目标是最大化该分数。
  - **进化搜索（FunSearch 改编）**：维护多个“岛屿”，每个岛屿存储候选 priority 函数。每步选择两个高适应度函数，由 LLM 建议改进版本。适应度由 8 个采样 KG 上的模型错误率估计。每轮包含多个 cycle，模型定期重训练加入新生成的困难实例。
  - **Auto-research 智能体**：仿照 Karpathy 的 autoresearch，使用 Claude Opus 4.6 单线程迭代修改 priority 函数，无需种群多样性，追求降低模型评估准确率且保持函数简洁。
  - **模型无关的 LLM 生成策略**：通过对话直接让 Claude 生成多种采样器（如 hill-climbing、backward chaining、motif-based、hybrid Atlas），可适用于任意 Datalog 规则集。
- **算法流程（文字说明）**：
  - 初始化：训练一个 Edge Transformer（ET）作为初始评估器。
  - 进化循环：对每个 round，进行多个 cycle，每个 cycle 更新 priority 函数，用该函数生成 KG，以 ET 错误率评估，选出最佳函数；重训练 ET 时合并历史生成的困难实例。
  - Auto-research 循环：每轮 agent 阅读上一轮结果，编辑 priority 函数，运行单轮生成、训练、评估，记录错误率。

## 6. 主要结论与发现

- **自动化采样有效**：三种策略（FunSearch 进化、Auto-research 智能体、直接提示）都能成功生成对 Edge Transformer 困难的小型 KG（≤8 实体）。

- **可迁移性**：实验在 LLM 生成的新世界（Iron Coast）同样成功，表明自动化闭环可推广。
- **进化函数趋同**：不同运行的 priority 函数在语义上相似（表1、表4），说明搜索收敛于相似策略。

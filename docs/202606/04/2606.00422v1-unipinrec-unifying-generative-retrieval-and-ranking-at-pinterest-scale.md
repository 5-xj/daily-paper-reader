## 摘要
现代推荐系统通常将检索和排序训练为独立模型，尽管两者都越来越依赖编码相同用户行为数据的大型Transformer，导致参数、计算和推理成本重复。先前的工作统一了模型架构，但未能统一完整流程：输入格式、训练程序和推理栈在各个阶段仍然碎片化。我们提出UniPinRec，在Pinterest上实现了检索和排序的全栈统一：一种输入格式、一个模型、一个训练阶段，并部署在现有推理基础设施中。共享的Transformer将用户动作序列编码为候选无关的表示，通过任务特定的头部分支到检索（ANN点积）和排序（交叉注意力）。三个关键思想使其成功：（1）掩码动作建模（MAM）消除了交织，使得无需加倍上下文长度即可实现权重共享；（2）混合训练示例将动作序列与feedview印象候选集配对，以同时满足两个目标；（3）跨阶段KV缓存共享重用检索中的用户历史计算用于排序，相比于独立部署两个模型，总FLOPs减少。在Pinterest核心页面部署后，UniPinRec带来了约+1%的在线参与度提升，同时将端到端推理延迟降低11.1%，QPS提升63.6%。据我们所知，这是首次在生产推荐系统中实现检索和排序的全栈统一，覆盖输入、模型、训练和推理。

## Abstract
Modern recommendation systems predominantly train retrieval and ranking as separate models despite both increasingly relying on large transformers encoding the same user behavior data, duplicating parameters, compute, and serving cost. Prior work unifies the model architecture but not the full pipeline: input formats, training procedures, and serving stacks remain fragmented across stages. We present UniPinRec, which achieves full-stack unification of retrieval and ranking at Pinterest: one input format, one model, one training stage, deployed within existing serving infrastructure. A shared transformer encodes the user action sequence into candidate-independent representations that branch into retrieval (ANN dot-product) and ranking (cross-attention) via task-specific heads. Three ideas make this work: (1) Masked Action Modeling (MAM) eliminates interleaving, enabling weight sharing without doubling context length; (2) Blended training examples pair action sequences with feedview impression slates to satisfy both objectives jointly; (3) Cross-stage KV cache sharing reuses user-history computation from retrieval for ranking, reducing total FLOPs versus serving two independent models. Deployed in the Pinterest core surfaces, UniPinRec delivers approximately +1% online engagement lift while cutting end-to-end serving latency by 11.1% and lifting QPS by 63.6%. To our knowledge, this is the first full-stack unification of retrieval and ranking, covering inputs, model, training and serving, deployed in a production recommendation system.

## 1. 论文的核心问题与整体含义

- **研究动机**：现代工业推荐系统通常将检索（retrieval）和排序（ranking）作为独立模型分别训练。尽管两个阶段都依赖大型 Transformer 编码相同的用户行为序列数据，却无法共享参数和计算，导致参数、训练计算和推理成本的大量重复。
- **问题定义**：先前工作（如 HSTU、OnePiece）只统一了模型架构，但输入格式、训练流程和服务基础设施仍然碎片化；端到端生成方法（如 OneRec）则完全替换了传统漏斗，难以与现有流水线兼容。
- **核心目标**：实现“全栈统一”——即输入格式、模型架构、训练和服务四方面的统一，并在生产系统中部署，以降低冗余并提升效率。
- **整体意义**：UniPinRec 是首个在生产推荐系统中实现检索和排序全栈统一的工作，验证了通过模型/系统协同设计可在不损失质量的前提下显著降低服务成本。

## 2. 方法论

- **核心思想**：在现有生成式检索模型 PinRec 的基础上，扩展出排序能力。使用共享 Transformer 将用户动作序列编码为候选无关表示，再通过任务特定头部分支为检索（ANN 点积）和排序（交叉注意力）。三个关键技术：
  - **Masked Action Modeling (MAM)**：将动作标签与物品嵌入在特征维度拼接，训练时随机掩码历史动作。避免了 HSTU 中动作/物品交织导致的序列长度加倍和输入格式不一致问题，实现了权重完全共享。
  - **混合训练样本与联合损失**：每个训练样本包含用户的过去动作序列（用于检索目标）和未来的完整印象候选集（含正负标签，用于排序目标）。联合损失 = 下一物品预测的采样 softmax 损失 + 各动作类型（点击、保存、隐藏等）的 BCE 损失，按权重平衡。
  - **跨阶段 KV 缓存共享**：检索阶段完成用户历史编码后，将其 KV 缓存通过 GPU 内存池直接共享给排序阶段。排序时仅需对候选 token 做注意力解码（O(nk)），避免重复 O(n²) 的计算。
- **其他关键技术**：
  - 注意力模式：候选位置之间相互屏蔽（M-FALCON 模式），使用 flex attention 实现稀疏化。
  - 数据基础设施：通过 Ray 在训练器内做桶连接（in-trainer join），避免离线数据重复。
  - 服务架构：检索和排序作为独立的 Triton 进程，通过 CPU 托管的 Faiss 索引完成 ANN 搜索，并使用 `search_and_reconstruct` 同时返回候选 ID 和嵌入，省去排序阶段重新编码候选。
  - FP8 混合精度训练和推理，使用 Transformer Engine 融合内核，以约 0.5% 离线指标下降换取效率提升。

## 6. 主要结论与发现

- **离线**：
  - UniPinRec 的检索 Recall@10 与生产检索模型 PinRec 持平（+0.2% 微小提升），说明联合排序训练不损害检索质量。

  - 联合训练（UniPinRec）优于仅排序训练（w/o item loss），证明检索目标对排序有正则化帮助。
- **在线**：
  - Board More Ideas：表面保存率 +0.95%，全站保存率 +0.08%。

## 7. 优点

- **全栈统一**：真正实现输入、模型、训练、服务四维统一，而不只是架构层面的共享。
- **兼容现有系统**：作为 drop-in replacement 部署，与下游排序器、混合器兼容，支持分阶段回滚和 A/B 测试。
- **创新技术**：
  - MAM 避免了序列长度膨胀和格式冲突，简洁有效。
  - 跨进程 GPU 内存池共享 KV 缓存，是系统与模型协同设计的亮点。
  - 数据构建使用 Ray 内训器连接，避免离线重复和版本管理问题。
- **在线收益显著**：不仅改善参与度，还降低延迟和提升吞吐量，同时证明统一模型可以胜过专业分离模型。

## 8. 不足与局限

- **实验覆盖**：
  - 未报告在更大规模候选集或不同领域（如搜索、广告）下的性能；结论可能受限于 Pinterest 的特定场景。
  - 缺乏与其他真正端到端生成方法（OneRec、OneRanker）的直接离线对比。
  - 未讨论冷启动用户或新物品的适应能力。
- **偏差风险**：
  - 在线实验仅用一个下游排序器（TransAct V2），统一模型的效果可能被下游稀释，实际收益可能被低估。
  - 稀疏动作类别（如隐藏）的权重调整主要靠手工调参，缺乏自适应机制。
- **技术局限**：
  - 当前仅统一了 L0+L1（检索与上层排序），L2 深度排序及校准部分尚未统一；作者计划后续扩展。
  - FP8 量化导致 0.5% 离线指标下降，在线实验中未采用 FP8 解码（因批处理形状约束导致延迟增加），限制了完全效率优势。
  - MAM 的掩码比例（0.2）在更大模型或更长序列上未必最优，未做更细致的搜索。
- **可复现性**：算力、训练细节、超参数（除掩码率外）未充分披露，对独立复现构成障碍。

## 摘要
符号回归（SR）从数据中发现紧凑的数学表达式，然而近期基于LLM的进化方法仍然样本效率低下，因为它们主要依赖标量反馈（如MSE）。我们识别出一个核心限制：现有方法将候选提议与搜索指导混为一谈，要求LLM仅从单一分数中推断如何进化表达式、诊断其错误并重用过往经验。为解决此问题，我们提出刻意进化（DE），一种将符号生成与搜索控制解耦的智能体框架。DE通过自适应算子引导LLM的提议以确定搜索方向，通过分析工具进行结构诊断，并通过反思记忆利用轨迹级经验。在LLM-SRBench上的实验表明，DE在多个科学领域一致优于代表性的基于LLM的SR基线方法，同时仅使用标准样本预算的40%。

## Abstract
Symbolic regression (SR) discovers compact mathematical expressions from data, yet recent LLM-based evolutionary methods remain sample-inefficient because they rely mainly on scalar feedback such as MSE. We identify a core limitation: existing methods conflate candidate proposal with search guidance, requiring the LLM to infer how to evolve an expression, diagnose its errors, and reuse past experience from a single score. To address this, we propose Deliberate Evolution (DE), an agentic framework that decouples symbolic generation from search control. DE guides LLM proposals with adaptive operators for search direction, analytical tools for structural diagnosis, and reflective memory for trajectory-level experience. Experiments on LLM-SRBench show that DE consistently outperforms representative LLM-based SR baselines across diverse scientific domains while using only 40% of the standard sample budget.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究领域**：符号回归（Symbolic Regression, SR）——从观测数据中发现紧凑、可解释的数学表达式，是科学发现的关键工具。
- **现有方法的瓶颈**：近期基于大语言模型（LLM）的进化SR方法，如LLM-SR、LASR、SGA，将LLM作为进化算子，但**样本效率极低**（通常每个问题需要评价10³个候选表达式）。根本原因在于：这些方法将**候选提议（proposal）** 与**搜索引导（search guidance）** 混为一谈，仅依靠单一标量分数（如MSE）来驱动LLM，却要求LLM同时推断出：如何修改表达式、为什么出错、如何复用历史经验——这远远超出单一分数的信息量。
- **缺失的三个关键信号**：
  - **方向**：缺少“该采用哪种符号操作”（细化/变异/交叉/重启）。
  - **诊断**：MSE无法定位错误源（如缺失周期项、变量耦合错误、量纲不一致）。
  - **记忆**：每个候选几乎独立，无法积累“哪些修改经常失败/哪些模式反复成功”。
- **本文贡献**：提出**Deliberate Evolution（DE）**——一个智能体框架，**显化解耦符号生成与搜索控制**，将搜索过程从“弱引导的试错”转变为“有明确意图、定位证据和历史经验的结构化科学搜索”。

## 2. 方法论：核心思想与关键技术细节

### 2.1 核心思想
DE在标准进化循环中，引入三类显式引导信号，替代纯MSE驱动：
- **方向性引导（Directional Guidance）**：通过自适应算子（Adaptive Operators）选择采用何种符号操作。
- **诊断性引导（Diagnostic Guidance）**：通过工具增强分析（Tool-Augmented Proposal）定位表达式结构缺陷。
- **历史性引导（Historical Guidance）**：通过反思记忆（Reflective Memory）总结轨迹级经验。

### 2.2 关键模块

#### (1) 自适应算子选择（Adaptive Operators）
- **算子集合** \(O = \{o_{\text{ref}}, o_{\text{mut}}, o_{\text{cross}}, o_{\text{reg}}\}\) 分别对应**细化**（保守编辑）、**变异**（结构大幅改变）、**交叉**（与精英表达式重组）、**重新生成**（完全从头开始）。
- **状态驱动选择**：根据父表达式 \(f_p\) 的归一化排名 \(\tilde{r}_p\) 和访问次数 \(v_p\) 将搜索映射到四个离散状态 \(s_t \in \{0,1\}^2\)。每个状态维护一个权重向量 \(w_s^{(t)}(o)\)，通过**归一化+探索底噪**形成分类策略 \(\pi_s^{(t)}\)，然后采样算子。
- **策略更新**：基于相对改进 \(r_t = \text{clip}\left(\frac{\ell(f_p) - \ell(f_t)}{\ell(f_p)+\varepsilon}, r_{\min}, r_{\max}\right)\)，采用**乘性更新**（成功增大权重，失败衰减，但有下限保护）。
- **逃逸停滞**：当连续h轮最佳损失无明显改进时，触发放宽选择、增大探索算子概率，并总结停滞轨迹到记忆。

#### (2) 工具增强提议（Tool-Augmented Proposal）
构建诊断报告 \(a_t\)，包含三个工具的输出：
- **数据探查器（Data Profiler）**：分析变量范围、算符域、相互作用统计、周期/奇异模式。
- **残差诊断（Residual Diagnostic）**：分析残差与变量变换的相关系数，定位缺失项（如周期项、平方项）或异方差性。
- **量纲验证器（Dimensional Verifier）**：递归检查表达式的物理单位一致性，过滤不合理结构。
这些工具将标量误差转化为局部结构反馈，注入LLM上下文，实现靶向修改。

#### (3) 反思记忆（Reflective Memory）
- **触发更新**：当周期性检查（每K轮）或出现突破性提升 \(\Delta_t > \epsilon_{\text{mem}}\) 时，触发记忆更新。
- **构建反思上下文**：包含当前问题、旧记忆、精英表达式、失败尝试、突破性编辑，LLM蒸馏出可重用的规则（如“周期性函数应该包裹sin/cos”）。
- **存储与压缩**：采用FIFO机制，保持内存简洁，保留成功模式和失败模式，删除冗余。

#### 算法流程（每轮t）
1. 从父种群通过Boltzmann采样得到父表达式 \(f_p\)。
2. **方向性引导**：计算状态 \(s_t\)，从策略 \(\pi_{s_t}^{(t-1)}\) 采样算子 \(o_t\)。
3. **诊断性引导**：用工具生成诊断报告 \(a_t\)。
4. LLM生成骨架 \(\tilde{f}_t \sim p_\theta(\cdot | Q, f_p, o_t, a_t, M_{t-1})\)。
5. **BFGS** 拟合常数得到完整表达式 \(f_t\)。
6. 更新种群、策略权重和记忆。
7. **停滞控制**：若停滞则进行全局逃逸操作。

### 2.3 效率分析（Hitting-Time）
假设引导使单步成功概率从 \(p_0\) 提升至 \(p_0 + \gamma\)，则击中目标所需样本数的失败概率从 \(\exp(-k p_0)\) 降至 \(\exp(-k(p_0+\gamma))\)，证明引导可指数级加速发现。

## 8. 不足与局限

1. **骨干模型范围有限**：仅测试了两个开源模型（8B和4B），未评估GPT-4、Gemini等更强或闭源模型。结论的泛化性可能受限于模型规模与架构。
2. **缺乏理论收敛保证**：DE本质上仍是启发式进化搜索，不能保证在有限步内找到全局最优。
3. **推理延迟较高**：每轮需要调用LLM生成、BFGS拟合，相比传统GP显著更慢，不适用于实时控制或极低延迟场景。
